# zone_reactor v2 — самообучаемый классификатор силы зон + трейдер-аргументация в TG

## Цель
Нейро-система, которая: (1) детектит ICT-зоны на 6h/8h/12h/D/3d; (2) самообучаясь,
классифицирует их на СИЛЬНЫЕ (дадут реакцию) / СЛАБЫЕ; (3) шлёт в Telegram сообщение
с аргументацией ЯЗЫКОМ ТРЕЙДЕРА: «BTC пришёл в зону 61000 — СИЛЬНАЯ (P=0.82): D-FVG +
проторговка 18 свечей здесь ранее + 6h-OB + 12h-FVG; против: близко к premium».

## Честный статус (откуда стартуем)
v1 показал: на МОИХ 18 фичах под чистым исполнением зоны разделяются слабо (AUC 0.54).
НО я НЕ пробовал ключевые фичи, которые ты назвал. Ставка v2 — РИЧЕ ФИЧИ + само­обучение,
а не другой алгоритм (CatBoost/NN дают тот же потолок на старых фичах). Дисциплина
валидации (см. ниже) — НЕОБСУЖДАЕМА, иначе получим «high-tech horoscope».

## БЛОК 1 — РИЧЕ ФИЧИ (главная ставка, то чего не было)
1. **Volume/time-acceptance в зоне** (твоя идея «проторговка свечей»):
   - сколько свечей ранее ЗАКРЫЛОСЬ внутри зоны (acceptance);
   - суммарный объём, торговавшийся в диапазоне зоны до касания (volume-at-price);
   - время, проведённое ценой в зоне (auction time);
   - сколько раз зона тестировалась и удерживалась.
2. **Multi-TF type-specific стек** (не просто count): one-hot «есть ли D-FVG / 6h-OB /
   8h-OB / 12h-FVG / 3d-OB ...» — конкретные комбинации ТФ×тип, чтобы сеть нашла, какие
   стеки сильны (D-FVG+12h-OB ≠ 6h-FVG+8h-FVG).
3. **ICT-контекст**: liquidity pools за зоной (equal highs/lows = цель реакции),
   sweep до зоны, premium/discount (fib OTE), сила displacement, создавшего зону,
   mitigated/unmitigated, breaker.
4. **Multi-TF индикаторы**: RSI/Hull/EMA на 6h/8h/12h/D/3d, дивергенции.

## БЛОК 2 — ЦЕЛЬ (лейбл) — чище, чем fixed +5%
Проблема +5%: коуплится с волатильностью/геометрией (артефакты v1). Варианты:
- «реакция в ATR-единицах» (≥2·ATR отскок до пробоя) — нормирует волатильность;
- «зона удержалась структурно» (реакция + не пробила за N баров) с реалистичным 1h-резолвом
  и входом по close 12h-свечи касания (твоя модель) С САМОГО НАЧАЛА;
- регрессия на величину отскока (не бинар) → ранжирование силы.

## БЛОК 3 — САМООБУЧЕНИЕ (как ты просишь — «гоняется пока не научится»)
1. **Hard-example mining + focal loss** — фокус на путаемых зонах.
2. **Walk-forward continual learning** — переобучение по мере прихода данных (адаптация
   к режиму; v1 показал, что оптимум зависит от режима).
3. **Error-driven feature search**: после каждого прогона кластеризуем ошибки (FP/FN),
   находим систематические провалы → добавляем фичу/вес → повтор. Цикл до плато OOS.
4. **Калибровка** (изотония) → P интерпретируема для порога/сайзинга.
5. **Стоп-критерий**: когда purged-CV OOS AUC перестаёт расти (честно — выше потолка
   данных не прыгнуть; «гоняться вечно» = переобучение).

## БЛОК 4 — МОДЕЛЬ
- Tabular (риче фичи): **CatBoost-GPU** (готов) / HGB — baseline.
- Sequence (если tabular упрётся): **PyTorch CUDA** (cu130) — 1D-CNN/Transformer на
  сырых multi-TF OHLCV-окнах → ищет «сложные закономерности» сам. Вот где GPU нужен.

## БЛОК 5 — ВАЛИДАЦИЯ (НЕОБСУЖДАЕМО — иначе обманем себя)
Пройдено ВСЁ, иначе не деплоим:
- purged + embargo CV (горизонт лейбла);
- строгий OOS train 2020-24 / test 25-26;
- **permutation null-test** (реальные зоны vs случайная разметка — поймал артефакт в v1);
- leak-hunt: фичи ≤ касания, реалистичное 1h-исполнение, нет fractal/width-тавтологии;
- per-year стабильность + net_R с издержками.
Бар: zones-only (без волатильности) purged-CV И OOS AUC > 0.58, net_R на 1h > 0, 0 плохих лет.

## БЛОК 6 — TELEGRAM-АРГУМЕНТАЦИЯ
Движок объяснений (explain.py, occlusion/SHAP-атрибуция) → фразы трейдера:
«Зона X СИЛЬНАЯ (P=..): [топ +факторы]; против: [топ −факторы]». Шлём при касании зоны
в live (скринер на 6h/8h/12h close).

## ПОРЯДОК РЕАЛИЗАЦИИ
1. **Блок 1 (риче фичи, особенно acceptance/volume-at-price)** — единственное реально
   новое; строим и валидируем ПРИРОСТ AUC над v1 (0.54). ← старт.
2. Если прирост есть → Блок 3 (самообучение) + Блок 2 (target) → Блок 4 (sequence/GPU).
3. Если прирост значим и проходит Блок 5 → Блок 6 (live TG).
4. Если acceptance-фичи НЕ дают прироста на честном OOS → это финальный честный ответ.
