# zone_reactor — нейро-модуль определения зон реакции (валидация: сигнала нет)

**Цель (запрос пользователя 2026-06-10):** нейро-проект, который по ТФ 8h/12h/D/3d
показывает сильные диапазоны, от которых ожидается реакция цены ≥5%; самообучаемый,
на GPU (RTX 4070 Ti Super).

## Постановка (правильная, без утечки)

Единица = **12h confirmed fractal** (swing low → ждём long-реакцию, high → short).
Фрактал — уже точка интереса, поэтому НЕ предсказываем «фрактал ли это» (тривиально из
формы свечи; именно это давало ложный AUC 0.93 в etap_170 — детекция фрактала, не сила
зоны). Предсказываем **УСЛОВНО**: даст ли фрактал реакцию ≥5% (зона сильная) или нет.

- **Лейбл:** reaction5 = цена прошла ≥5% от экстремума в сторону разворота за 20×12h
  раньше, чем пробила экстремум. Train 2020→2024, test 2024→2026 (OOS).
- **Фичи силы зоны (`features.py`):** multi-TF OB+FVG confluence (8h/12h/1d/3d) на
  экстремуме-фитиле, HTF-вес + untouched; ICT (liquidity sweep, premium/discount,
  reclaim); индикаторы (RSI/EMA200/Hull/ATR/vol-z/BB%b); HTF тренд 1d/3d. Всё ≤ подтв.

## Результат валидации — СИЛА ЗОНЫ РЕАКЦИЮ НЕ ПРЕДСКАЗЫВАЕТ

`baseline.py`: OOS AUC 0.640, purgedCV 0.635 — выглядело как сигнал. НО `diagnostic.py`:

| Группа фич | OOS AUC |
|---|---|
| ВСЕ (20) | 0.620 |
| **только atr_pct (1)** | **0.613** |
| только зоны (7) | **0.485** (ниже монетки) |
| зоны+ICT (без vol) | 0.541 |
| per-asset зоны+ICT | BTC 0.555 / ETH 0.560 / SOL 0.492 |

**Вывод:** AUC 0.64 — это `atr_pct` = ВОЛАТИЛЬНОСТЬ («хватит ли рынку волатильности на
фиксированные 5%»; поэтому SOL реагирует 82%, BTC 61%). Multi-TF сила зоны (8h/12h/D/3d
confluence) реакцию НЕ предсказывает — зоны-только AUC 0.485, per-asset ~0.5. Третье
независимое подтверждение (см. etap_192: zone-strength на Bulkowski 0.51, не монотонна).

**Глубокий смысл:** «сильная зона» (ICT) указывает ГДЕ вероятен разворот (99% фракталов
у зон), но НЕ предсказывает РАЗМЕР реакции. Follow-through определяется тем, что приходит
ПОСЛЕ (моментум/поток) → это путь сделки (etap_193, in-process AUC 0.58 — единственный
реальный сигнал), а не пред-входная сила зоны.

## GPU / самообучаемая сеть — НЕ построена сознательно

torch установлен (CPU; CUDA cu124 заблокирован SOCKS-прокси на pytorch-индексе). GPU-сеть
с самообучением НЕ строилась: при zones-only AUC 0.485 учить нечего — глубокая сеть лишь
переоткроет тривиальное «высокая волатильность → 5% чаще» или переобучится. Строить GPU-DL
на волатильность-артефакте = «high-tech horoscope» (López de Prado). Если строить —
только при доказанном separable-сигнале (purgedCV И OOS > 0.58 на фичах БЕЗ atr_pct).

## ВАЛИДИРОВАННАЯ КОНФИГУРАЦИЯ (после критики ширины зоны)

Критика пользователя: широкие «зоны» (10-20%) геймят лейбл held (реакция +5% раньше
закрытия за дальним краем тривиальна, если край на 15% дальше). Подтвердилось: топ-P
зоны были 11-20% шириной = мусор. ФИКС — кап ширины ≤3% (реалистичные ICT-зоны):

| макс.ширина | AUC | P≥0.6 R/tr | per-year (2024/25/26) |
|---|---|---|---|
| ≤2% | 0.632 | +2.155 (n=51) | — |
| **≤3%** | **0.671** | **+1.396** (n=151) | **+1.39 / +1.31 / +1.74 (все плюс)** |
| ≤5% | 0.704 | +0.671 | +0.76/+0.71/+0.26 |
| все (с мусором) | 0.728 | +0.586 | (мусор разбавляет R) |

→ **Ядро модели: zone_width ≤3% + held/broke GBM + фильтр P≥0.6 → +1.4 R/tr, ~65/год,
стабильно по годам.** Пережило: 2 фикса утечек (intrabar stop, last-closed фичи) +
критику ширины + per-year. Драйверы (НЕ multi-TF confluence): RSI/premium-discount/
свежесть/волатильность. `zone_width_sanity.py` — этот анализ.

## Файлы
- `features.py` — модуль силы зоны (multi-TF + ICT + индикаторы), переиспользуем.
- `baseline.py` — gating-валидация (sklearn, purged-CV + OOS + монотонность).
- `diagnostic.py` — сила зоны vs волатильность (поймал артефакт).
- `dataset.csv` — 3437 фракталов BTC+ETH+SOL с фичами и лейблом.
