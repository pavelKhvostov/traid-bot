# ASVK Power Zone — Knowledge Vault

Граф знаний проекта. При старте сессии Claude Code читает этот файл и [[текущие приоритеты]],
а затем переходит к нужным разделам по ссылкам.

## Главное

- [[архитектура проекта]] — 5 слоёв, диаграмма, модули.
- [[стек и зависимости]] — Python, Binance, Telegram, pandas.
- [[структура CSV]] — схемы файлов `data/` и `signals/`.

## Стратегии (по одной заметке на каждую)

- [[s1 obx4 + ob1h]] — OBx4-цепочка на старшем ТФ → первый OB1h.
- [[s2 ob htf + ob1h]] — OB на 2h-3d → первый OB1h.
- [[s3 rdrb + ob1h]] — RDRB-зона → первый OB1h.
- [[s4 снятие фрактала]] — снятие HH или LL.
- [[s5 fvg + ob1h]] — FVG на 2h-3d → первый OB1h.

## SMC-примитивы

- [[что такое order block]] — логика OB, bullish и bearish.
- [[что такое fvg]] — Fair Value Gap, три свечи.
- [[что такое rdrb]] — Rally-Drop-Rally-Base.
- [[что такое обx4 цепочка]] — 5 свечей, красная-зелёная-красная-зелёная + FVG.
- [[фракталы билла уильямса]] — HH и LL по i±2.
- [[правило первого OB после возврата]] — ключевое правило всех 4 "возвратных" стратегий.

## Принятые решения

- [[почему csv а не postgres]] — MVP, простота, быстрый старт.
- [[почему только первый ob после возврата]] — уменьшает ложные срабатывания.
- [[почему binance а не bybit]] — WS стабильнее, исходник `obxxx.py`.
- [[почему только btc eth sol]] — ликвидность, совпадает с `obxxx.py`.
