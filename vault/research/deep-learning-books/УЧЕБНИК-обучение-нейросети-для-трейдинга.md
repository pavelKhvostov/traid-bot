# 📘 УЧЕБНИК: как грамотно обучать нейросеть для трейдинга (синтез 3 книг)

Синтез ПОЛНОГО чтения 3 книг (Goodfellow 801 стр, Nielsen 293, Nikolenko 477 — 10 агентов прочитали все главы) под НАШУ задачу: нейросеть на табличных фин-фичах, мало данных, сильный overfit (train ρ 0.77 / test 0.09), низкий signal-to-noise. Только применимое.

## 🎯 ДИАГНОЗ нашей проблемы (все 3 книги согласны)

train ρ 0.77 / test ρ 0.09 = **учебниковая правая ветвь U-кривой capacity** (Goodfellow 5.2): модель слишком ёмкая для данных, запоминает шум. Лечение — НЕ больше эпох/слоёв, а: **уменьшить эффективную ёмкость + расширить данные + сильная регуляризация + честная валидация.**

## ⭐ ТОП-ПРИЁМЫ (сошлись 3+ агента = делать обязательно)

### 1. Data augmentation / feature-masking / denoising (Nielsen ×2, Nikolenko ×2, Goodfellow 7.4-7.5)
**Главное лекарство от overfit на малых данных.** Агрессивно зашумлять/обнулять 10-30% входных фич на каждом шаге (denoising) + input noise injection. Сеть учится не полагаться на отдельные шумные фичи. У нас noise injection есть (etap_181), но **feature-masking (random dropout входов) — НЕ пробован**.
→ Применить: `x = x * bernoulli_mask(p=0.15) + noise`.

### 2. Сильный baseline (LogReg/GBM) как ПОЛ (Goodfellow 11.2, 5.3)
**Если MLP не бьёт LogReg/GBM — сеть переусложнена, не нужна.** Обязательный sanity. У нас LightGBM был ρ~0.11 (BTC), MLP 0.09 — то есть **GBM не хуже сети**. Книги: на табличных малых данных это норма, не позор.
→ Применить: всегда сравнивать с LogReg + GBM. Если не бьём — упрощать сеть или использовать GBM.

### 3. weight decay БЕЗ bias/BatchNorm/LayerNorm (Goodfellow 7.1, Nielsen)
L2 применять ТОЛЬКО к весам, НЕ к bias и параметрам нормализации. Мы decay'им всё → неверно.
→ Применить: param groups в AdamW — `weight_decay=0` для bias/norm.

### 4. LayerNorm вместо BatchNorm (Nikolenko гл.4)
Малые данные → маленькие батчи → **шумные BN-статистики дестабилизируют** (наши фолды etap_181 были нестабильны: train ρ 0/0.4!). LayerNorm не зависит от батча.
→ Применить: заменить nn.BatchNorm1d на nn.LayerNorm. ИЛИ SELU + AlphaDropout (self-normalizing).

### 5. L1 на входной слой = отбор шумовых фич (Nikolenko ×2, Goodfellow 7.10)
Низкий SNR + много hand-engineered фич → L1 на первом слое обнуляет бесполезные. Встроенный feature selection.
→ Применить: L1-penalty на weights первого Linear.

### 6. Label smoothing 0.05-0.1 (Nikolenko, Goodfellow)
Против overconfidence на шумных метках. Мягкие таргеты.
→ Применить: в CORAL/CE сглаживать таргеты.

### 7. Систематический поиск гиперпараметров (Nielsen, Goodfellow 11.4)
**Random search > grid** (Goodfellow). Objective = усреднённый по Purged-фолдам test ρ. Optuna-TPE.
→ Применить: Optuna по dropout/wd/hidden/lr/noise, objective = mean CV ρ.

## 🔬 ПРИЁМЫ ВТОРОГО ЭШЕЛОНА (1-2 агента, попробовать)

- **SWA / Polyak-EMA усреднение весов** (Goodfellow 8.7.3) — усреднение траектории сглаживает, дёшево, НЕ используем.
- **Unsupervised pretraining / автокодировщик** (Goodfellow 7.6, Nikolenko 5.5) — много сырых баров, мало размеченных: предобучить нижние слои autoencoder'ом на ВСЕХ барах, потом дообучить. Прямой ответ на «мало данных».
- **MC-Dropout / variational dropout** (Nikolenko 10.5) — обучаемая интенсивность dropout + неопределённость на инференсе (можно фильтровать неуверенные).
- **Multi-task** (Goodfellow 7.7) — учить совместно качество + направление разворота + вспом. таргет (общий ствол регуляризует).
- **Heteroscedastic выход** (Goodfellow 6.2.2.4) — предсказывать и значение, и уверенность (precision) через гауссов NLL: при шуме модель сама взвешивает.
- **gradient clipping + LR warmup** (Goodfellow 8.2-8.3) — стабильность.
- **mini-batch size как гиперпараметр** (Goodfellow 8.1.3) — малый батч = регуляризатор.
- **reject-option / precision@coverage** (Goodfellow 11.1) — метрика «отказ от неуверенных»: для бота слать только сверх-уверенные.
- **каскад классификаторов** (Goodfellow 12.1.5) — дешёвый high-recall фильтр → точный high-precision.

## ⚙️ ЧТО УЖЕ ПРАВИЛЬНО (книги подтвердили)

Dropout ✓, early stopping ✓, AdamW ✓, ансамбль ✓, GELU (не насыщается) ✓, focal/CORAL (≈cross-entropy, нет slowdown) ✓, Purged K-Fold ✓, баланс классов ✓, noise injection ✓.

## 🧭 ПЛАН ГРАМОТНОГО ПЕРЕОБУЧЕНИЯ (по книгам)

1. **Данные: расширить до 2017** (вдвое больше — прямой ответ Goodfellow 11.3 «больше данных при overfit»). ✅ качается.
2. **Сильный baseline:** LogReg + GBM на тех же данных. Цель MLP — побить их.
3. **Анти-overfit сеть:** LayerNorm (не BN) + feature-masking augmentation + L1-вход + label smoothing + weight-decay-без-bias/norm + SWA.
4. **Honest валидация:** Purged K-Fold, train ρ vs test ρ (gap = диагноз), shuffle-тест.
5. **Optuna** по гиперпараметрам, objective = mean CV ρ.
6. Если MLP не бьёт GBM на бОльших данных → честно: GBM лучше для табличных, нейросеть не нужна для ЭТОЙ задачи (но нужна для разворотов etap_177).

## ⚠️ Честный мета-вывод книг

Все 3 книги сходятся: **на табличных малых шумных данных нейросеть редко бьёт GBM**, и главный рычаг — НЕ архитектура, а данные + регуляризация + честная валидация. Если после ВСЕГО этого test ρ не растёт — это Bayes error задачи (исход сделки = шум), а не плохое обучение. Тогда нейросеть направить на то, где сигнал есть — предсказание РАЗВОРОТА (etap_177, AUC 0.93).
