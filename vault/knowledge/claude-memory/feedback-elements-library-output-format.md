При запросе пользователя «**элементы библиотеки**» (или «покажи элементы», «какие элементы», «список элементов», «инвентарь зон», «что в `elements/`») использовать **строго 5-секционный формат**, зафиксированный как Правило 10 в `~/smc-lib/rules.md`.

**Why:** Пользователю нужен быстрый восстанавливаемый инвентарь библиотеки. 2026-05-29 я выдал именно такой формат, пользователь его утвердил как канон и попросил применять во всех будущих ответах. Унификация — чтобы каждый раз картина одинаковая, без когнитивной нагрузки на интерпретацию.

**How to apply:** Когда триггер срабатывает — НЕ упрощать, НЕ менять порядок секций, НЕ пропускать колонки. Источники истины: `ls ~/smc-lib/elements/*/`, заголовки definition.md, `grep "^def " code.py`, `ALL_TYPES` в `zones.py`. Класс зоны определять по [[zone-class-liquidity-inefficiency-block]].

## 5 секций (порядок строгий)

1. **Главная таблица** — **сгруппирована по классу** в порядке цикла Правила 8: `⛽ Liquidity → 🧲 Inefficiency → 🎯 Блок → Composite`. Каждая группа — отдельная подтаблица. Колонки: `Слаг | Заголовок | Свечей | Mitigation | Геометрия | Направление`. Composite-группа имеет +1 колонку `Композит классов`.
2. **Структура папки** — code block с `definition.md + code.py`
3. **Сигнатуры детекторов** — таблица `Элемент | Сигнатура | Возврат`
4. **Что в prediction-algo** — цитата `ALL_TYPES` + упоминание ob_vc-skeleton
5. **Не-элементы** — ссылки на `patterns/`, `vc/`, `indicators/` (но не элементы)

## Ключевые две оси (зафиксировано 2026-05-29)

- **`Геометрия`** = `range` (диапазон `[lo, hi]`) или `level` (точечное значение). **Главная различающая ось.** Фрактал + марубозу = `level` (за диапазоном не знаем где разворот); все остальные = `range`.
- **`Направление`** = всегда **`long / short`** (трейдинговое ожидание). Внутренние code-метки могут быть `high/low` (fractal) или `top/bottom` (RB) — в display унифицируется.

## Mapping internal → display direction

| Code-метка | Display | Где |
|---|---|---|
| `bottom` | `long` | RB |
| `top` | `short` | RB |
| `low` | `long` | fractal (FL sweep → long setup) |
| `high` | `short` | fractal (FH sweep → short setup) |
| `long` / `short` | без изменений | OB, FVG, RDRB и др. |

## Что НЕ включать

- Историю изменений (это в session-notes)
- Подробную геометрию зон (это в `zone_of_interest.md`, ссылаться)
- Trading strategy / entry / SL (это уровень патернов, не элементов)
- Композитные элементы (ob_liq, marubozu, ob_vc) → в отдельной группе Composite, не в основных трёх классах. Их формула композита раскрывается в колонке «Композит классов».
