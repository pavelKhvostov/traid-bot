**Marubozu — это аномалия, а не зона support/resistance.**

**Цель цены = уровень `open`** (точечный price magnet), не вся область тела. Тело marubozu — геометрическое описание свечи, **не "зона имбаланса для заполнения"**: внутри body (от open до close) проторговка УЖЕ была — импульс прошёл через эти цены. Имбаланс — это **отсутствие двусторонней проторговки на уровне open**: ни одной точки на противоположной от направления свечи стороне от open в течение бара.

- **LONG marubozu** (`open == low`): аномалия = ноль точек ниже open. Магнит = `open` (= `low`). Цена стремится вернуться к этому уровню, чтобы свеча "стала нормальной" (с проторговкой ниже open извне).
- **SHORT marubozu** (`open == high`): зеркально. Магнит = `open` (= `high`).

**Правильная последовательность взаимодействия:**
1. Marubozu формируется → возникает аномалия на уровне `open`.
2. Цена с высокой вероятностью возвращается к **`open`-уровню**.
3. Только **после касания open** возможен разворот в сторону исходного импульса.
4. Вход — на возврате к open-уровню, не на формирование свечи и не на середину body.

**Класс зоны** (по таксономии [[zone-class-liquidity-inefficiency-block]]): **inefficiency** (как FVG), но точечная — линия `open`, а не интервал.

**Why:** Двухступенчатое уточнение пользователя 2026-05-24:
1. "марубозу это аномалия. цена стремится исправить её. Это как гэп. Нужно заполнить и после взаимодействия развернуться уже можно".
2. "тело не есть имбаланс. само открытие есть цель. Цена хочет добраться до открытия и как бы сделать нормальную свечу с двумя проторгованными областями".

**Соотношение с geometric zone в smc-lib:**
- `elements/marubozu/code.py` хранит `zone = [body_bottom, body_top]` — это **геометрия** свечи (для рендера, для пересечений с другими зонами и т.п.).
- **Trading-таргет** = одна из границ body, совпадающая с open: `body_bottom` для LONG (open=low), `body_top` для SHORT (open=high).

**Чего НЕ делать:**
- НЕ говорить "цена заполняет тело" — тело не имбаланс.
- НЕ говорить про marubozu "S/R" / "ждём отскок при первом касании body".
- НЕ ставить вход в середине body — entry trigger именно на возврате к **open-уровню**.
- НЕ путать с RB (rejection = "пробовали и отказались" → S/R). Marubozu = "не пробовали с противоположной стороны от open" → imbalance, **противоположная** логика отработки.

**How to apply:**
- Trading-таргет = open level (одна цена).
- Entry trigger на возврате к open, expected reaction = продолжение исходного импульса.
- В рендерах/иллюстрациях marubozu выделять **open level отдельной линией**, body может быть фоном.
