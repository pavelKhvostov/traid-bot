При запросе пользователя «экспертный график» / «представить экспертный график» / «expert chart» — НЕ строить с нуля, а запускать каноничный скрипт `~/smc-lib/expert/chart.py`. Канон композита зафиксирован в `~/smc-lib/expert/chart.md`.

**Why:** Пользователь сказал 2026-05-28 «сохраняй этот алгоритм в библиотеку в экспертный график. И когда я попрошу представить экспертный график делаешь все для такой картинки.» После долгой итеративной настройки (chart_format база + Pred-12h overlay + HMA 12h/D + VWAPs эффективные/проработанные) этот композит признан финальным. Воссоздавать его с нуля каждый раз — пустая трата контекста.

**How to apply:**
1. Триггеры: «экспертный график», «представить экспертный график», «expert chart», «expert_chart», слова про «экспертный» в контексте визуализации.
2. Запустить: `python3 ~/smc-lib/expert/chart.py` (он сам докачает 1m данные через `fetch_btc_1m_missing.py`).
3. Output: `~/Desktop/i-rdrb-charts/btc_6h_pred12h_basket_<YYYY-MM-DD>.png` (дата = сегодня).
4. Дать пользователю путь и **краткий комментарий**:
   - Ключевые VWAP уровни (эффективные support/resistance), их значения и дистанция до цены
   - Ближайшая HMA (тренд)
   - Свежие pivot'ы за последние 24h (если есть)
5. НЕ редактировать скрипт без явного запроса пользователя.
6. Канон спецификации — [[../smc-lib/expert/chart.md]]. Зависимости — Правила 6, 7, chart_format.md, проект pred12h.

**Триггер «экспертное заключение»** / «куда пойдёт цена» / «дай мнение» — отдельный канон: `~/smc-lib/expert/opinion.py --tfs W,D,12h,4h,1h,15m` + синтез шагов 6-10 из [[../smc-lib/expert/opinion.md]].
