# 12h фракталы: OR-basket вместо AND-cascade (после F1∩F2∩F3)

Утверждено пользователем 2026-05-26.

## Правило

Начиная **поверх baseline F1∩F2∩F3** (1 266 / 619 conf / 18/18 imp), все дополнительные условия — **параллельные**:

```
good_basket = F1∩F2∩F3 candidate satisfies (C_a OR C_b OR C_c OR ...)
```

Каждое условие проверяется отдельно. Кандидат попадает в «корзину хороших», если совпало **хотя бы одно** условие.

## Why

- AND-cascade (F4∩F5∩…) терял important (F4 терял #4, #9 → 16/18; F5 ещё 3 → 13/18). Невосстановимо без OR.
- OR-basket гарантирует **recall 18/18**: достаточно подобрать условия так, чтобы каждый imp ловился хотя бы одним.
- Каждое условие оценивается **независимо** — свой keep/conf/P/imp_caught. Можно добавлять / убирать без переписывания.

## How to apply

### Шаблон стат-таблицы

| Условие | keep | conf | not conf | P(W) | Δ vs base | imp_caught/18 |
|---|---:|---:|---:|---:|---:|---:|
| (baseline) F1∩F2∩F3 | 1 266 | 619 | 647 | 48.9% | — | 18/18 |
| C_a (например F4) | … | … | … | … | … | … |
| C_b (например COUNTER VC any) | … | … | … | … | … | … |
| **Union (C_a ∪ C_b ∪ …)** | … | … | … | … | … | **должно быть 18/18** |
| Missed by union | base − union | … | … | … | … | 0 (требование) |

### Принципы
- Условия НЕ могут эксклюзировать pivots (это противоречит цели recall).
- Если предлагаемое условие при добавлении понижает P(union) — оно «худший» сигнал и стоит отказаться, не добавлять.
- Если imp не ловится ни одним условием — нужно добавить условие специально под него (анализ потерянных).
- Финальная корзина = union; precision корзины = conf/keep, всегда сравнивать с baseline 48.9%.
