# Prefill-silent при старте — маркируем сегодня как sent без рассылки

## Решение

При старте `Scanner.startup()` после догрузки свечей вызывает `_prefill_today_signals`,
который проходит по всем `(symbol, strategy, source_tf)` за последние 48 часов и для
каждой зоны с `trigger_time >= today_start_UTC` вызывает `mark_sent` с
`source: "prefill_silent"`. **Broadcast НЕ происходит.**

## Где живёт

- [scanner.py:67](../../../scanner.py#L67) — вызов `_prefill_today_signals`
- [scanner.py:69-130](../../../scanner.py#L69-L130) — сама функция
- `mark_sent` в [state.py](../../../state.py)

## Зачем

Если бы prefill отправлял сигналы, при каждом рестарте бот заваливал бы подписчиков
лавиной «сегодняшних» подтверждений, которые уже произошли в прошлом. С другой стороны,
**не помечать** их означает риск повторно отправить ту же сегодняшнюю зону, когда
закроется следующая 1h.

`prefill_silent` решает обе проблемы:
- В `state/sent_signals.json` ключ уже есть → `was_sent(k) == True` → диспатч молча скипнет.
- В Telegram в момент старта тишина.

## Граничные случаи

- **Окно 48 часов** ([scanner.py:75](../../../scanner.py#L75)) — берём с запасом, чтобы
  не пропустить вчерашние зоны, чьё подтверждение могло прийти на сегодняшней 1h.
- **`today_start = floor("D")` по UTC** — рубеж сегодняшнего дня, всё что раньше — не наш
  «сегодня».
