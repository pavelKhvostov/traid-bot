# Почему Binance, а не Bybit (или другой)

## Решение

Binance Spot — единственный источник OHLCV-данных и WebSocket-потока для бота.
Bybit/OKX/Coinbase не используются.

## Причины

- **Стабильность WebSocket.** В исходнике (`reference/obxxx.py`, на котором проект
  основан) WS-поток был на Binance; перепроверять стабильность другого провайдера
  и переписывать `update_df_incrementally` не было задачи.
- **Публичный API без ключей.** Binance Spot отдаёт klines REST + WS без аутентификации.
  Это укладывается в требование «не хранить и не запрашивать секреты бирж».
- **Ликвидность совпадает с задачей.** Сигналы строятся по BTCUSDT/ETHUSDT/SOLUSDT —
  на Binance Spot это самые ликвидные пары, проскальзывание/гэпы минимальны и логика
  стратегий (FVG, OB) валидна.
- **Существующая совместимость кода.** `data_manager.py` использует
  `https://api.binance.com` и `wss://stream.binance.com:9443/stream`. Любая другая
  биржа потребовала бы переписать формат свечей и поля.

## Что НЕ играло роли

- Комиссии — бот никаких сделок не делает, только сигналы.
- Объём торгов — бот объём не учитывает (см. [[что такое order block]]).
- Регуляторика — бот не торгует.

## Когда пересматривать

Если Binance API окажется недоступен (геоблок, отказ API), мигрировать на Bybit Spot —
формат свечей совместим, потребуется адаптация `data_manager.fetch_*` и URL WebSocket.
