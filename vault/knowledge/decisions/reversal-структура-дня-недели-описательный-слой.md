# Разворот дня/недели — описательный слой модуля аналитики (etap_255)

Запрос: усилить модуль аналитики более грамотным ОПРЕДЕЛЕНИЕМ разворотной точки дня
и недели. Раньше модуль только красил часовые бары по режиму day-type (тренд/боковик)
и НИКОГДА не помечал саму точку разворота и не отличал «разворотный день» от тренда.

## Что построено
- `research/daily_engine/etap_255_reversal_structure.py` — детектор + честное forward-
  исследование (3 монеты × 7 лет, per-year + base-rate).
- `reversal.py` (корень репо) — live-модуль: `classify_day` (форма дня + пивот, есть
  флаг developing для незакрытого дня), `weekly_structure` (свип PWH/PWL), RU-описания.
- Интегрировано (тест-контур): чарт `etap_225` (маркер ▲/▼ «разворот» на пивоте +
  бокс «форма дня»), caption `etap_227` (строки разворота), `signal_context` (строки в
  блок контекста сигнала). Тесты `tests/test_reversal.py` (11, всего 100 зелёных).

## Формы (описательно, на базе IB-фреймворка etap_217)
ДЕНЬ (1h): rev_up / rev_down (снят утренний экстремум + большой встречный фитиль +
закрытие на другой стороне), trend_up/trend_down, range. Пивот = HoD/LoD соответствующий.
НЕДЕЛЯ (1d→W): wk_sweep_high/low (сняли PWH/PWL и вернулись за уровень — недельный
разворот/свип), wk_cont_up/down (пробой удержан), wk_inside.

## ЧЕСТНЫЙ вердикт (адверсариальная проверка, агент)
**Направление ПОСЛЕ любой разворотной структуры = монетка** (dir_ok 42–58%,
год-нестабильно, ни одной кросс-ассетной год-стабильной ячейки). Стена «направление =
монетка» держится снова. → НЕТ прогноза направления, НЕТ ставок.

Две ловушки, пойманные при проверке:
1. **fill% (свип 52–73% vs продолжение 5–17%) — геометрический КОНФАУНД**, не поведение:
   `forward_stats` берёт для свипа СОСЕДНИЙ только что снятый уровень, а для продолжения —
   ДАЛЬНИЙ противоположный (медиана дистанции свип ~0.3–0.5 vs продолж ~1.1 ширины недели,
   ~3×). Сравнение невалидно.
2. **«Свип-уровень = магнит» — УБИТ** при контроле близости (etap_255b): на одинаковой
   дистанции свип-недели возвращаются к уровню НЕ чаще, а чуть РЕЖЕ немечёных
   (Δ touch1w: BTC −10.7пп, ETH −9.3, SOL −7.7). Высокий сырой revisit свипов — это
   близость, не «липкость».

Lookahead-аудит: чисто — `classify_day` берёт только бары завершённого периода;
forward строго вперёд (`d_idx>date`, `wk.index>date`); недельный PWH/PWL через shift(1).

## Решение
Слой едет в продакшн модуля **только как ОПИСАТЕЛЬ**: где развернулось (пивот), какой
формы день/неделя, какой уровень снят. Без directional-claim — в духе дашборда
(«это состояние дня, а не прогноз»). Это и есть «грамотнее определять разворот» =
лучшая ДЕТЕКЦИЯ, не предсказание.

Ценность подтверждена примером: 2026-06-10 BTC режим = ROTATION (плоско-серый), а слой
поймал «разворот ВНИЗ, пивот 15:00» — расхождение формы с режимом, которое старая
заливка теряла. Caption-строки молчат на трендовых/боковик/внутри-недели (не дублируют
режим) — появляются ТОЛЬКО при реальном развороте/свипе.

## Что НЕ делаем (стены)
- Не выдаём P(разворота) / направление — стена.
- Не используем fill%/магнит как edge — конфаунд/убито.
- Bulkowski-паттерны (etap_172, 13 детекторов) как direction-фича = стена (lookahead
  bulk_side был пойман: AUC 0.628→0.512). Здесь они не нужны — мы детектируем форму
  ЗАВЕРШЁННОГО периода, а не предсказываем.
