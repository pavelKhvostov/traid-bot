# Криволинейные паттерны (арки/округления) + открытие паттернов из данных

Запрос юзера (фото: плавная ДУГА-купол над снижением + отскок): проанализировать паттерн, научить
модуль его детектить, и провести глубокий анализ движений рынка чтобы вывести ещё паттерны.

## 1. Детектор кривизны — `research/ta_laws/curves.py` (тесты 6/6)

Признак арки = КРИВИЗНА, не наклон. Фит параболы `close ~ a·t² + b·t + c` на окне:
- знак `a`: a<0 = **купол ROUNDING_TOP** (фото юзера), a>0 = **чаша ROUNDING_BOTTOM** (cup);
- `r2_quad` (парабола описывает окно), `arc_gain = r2_quad − r2_lin` (дуга лучше прямой),
- `sagitta_atr = |a|·L²/8 / ATR` — физический прогиб дуги от хорды (порог 1.5 ATR = реально изогнута).
arm = i1 (правый край), каузально. `find_arcs` = скан окон 20/30/45/65 + дедуп по силе/пересечению.
Сверка на live BTC: чисто обвёл чашу/J-launch 06-13→06-15. Купол ловится симметрично.

## 2. Закон арок — `arc_analysis.py` (2834 арки, BTC/ETH/SOL, 1h→D, triple-barrier ±1.5ATR)

**ГОЛАЯ АРКА ≈ НЕЙТРАЛЬНА** (rev_R +0.030, null +0.010, p=0.129 — null НЕ бьёт). Форма сама = фольклор,
как и вся стена проекта. Купол-со-спуском (точь-в-точь фото) сам по себе: P(отскок) 51%, p=0.13 = монетка.

**НО есть УСЛОВНЫЙ закон mean-revert** (после завершения дуги цена разворачивается):
- **apex (зрелость хода в i1) — доминирующий разрез**: apex рано <0.4 → rev_R **−0.174** (продолжение, 0/3) ↔
  apex поздно >0.7 → **+0.109** (разворот, **3/3**, p=0.004), центр 0.4-0.7 → +0.074 (3/3, p=0.005). Спред 0.28.
- **изогнутость**: средняя sagitta 2.5-4 ATR → rev_R +0.107 (**3/3**, p=0.008); слабая <2.5 → ноль.
- **против контекста** (mtf 0-1/3) → разворот 3/3; **по контексту** → продолжение (−0.044).
- крупная дуга depth≥8 ATR → +0.051 (3/3, p=0.063). Докуда отскок: MFE медиана ~3 ATR (MAE ~2.6).
Итог: **хорошо изогнутая арка с apex в центре/поздно, идущая против контекста → mean-revert** (3/3).
КАВЕАТ: apex≈наклон в i1 (2a·L·(1−apex_pos)) → эффект частично «крутой спуск продолжается / пологий
разворачивается» = моментум-конфаунд; нужен momentum-matched null чтобы изолировать вклад именно дуги.

## 3. Открытие паттернов ИЗ ДАННЫХ — `shape_discovery.py` (17279 окон, KMeans K=16 на z-форме)

Нарезка окон 24 бара → z-score+ресэмпл 20 точек → кластеры форм → исход triple-barrier next-24 + null +
cross-asset + год. **2 робастных паттерна** (|edge|>0.06, p<0.05, sym 3/3):
- **кл10 BULL: «провал + резкий рывок вверх» (J-launch / higher-low пружина)** → ВВЕРХ (edge +0.095, p=0.003, год 5/7).
- **кл13 BEAR: перегретый/устойчивый рост → ВНИЗ** (edge −0.086, p=0.002, год 6/7).
Кандидаты (3/3, p<0.15): **кл9 blow-off (выброс вверх → вниз)**, **кл12 КУПОЛ ∩ → вниз**, кл11 J-launch, кл4 рост→вниз.
**Открытие НЕЗАВИСИМО переоткрыло «купол/blow-off истощение → вниз»** (кл12/кл9) = подтверждение arc-закона
другим методом. Асимметрия «рост→вниз» в нескольких кластерах = mean-reversion ралли на этих активах/периоде.
КАВЕАТ: окна перекрываются 50% → autocorrelation раздувает n; доверять cross-asset 3/3 + год, не p; edge на
симметр. барьере, не PnL с костами.

## Синтез
Семейства криволинейных паттернов: **(BEAR) истощение вверх — купол/blow-off/перегретый рост → разворот вниз**;
**(BULL) J-launch/higher-low пружина → продолжение вверх**. Голая форма не работает; edge = форма + условия
(изогнутость + зрелость/apex + контекст) — тот же принцип, что [[ta-pattern-taxonomy-direction-vs-extent]]
(куда=контекст, форма — модулятор). Монетизация (entry/stop/TP/косты) + momentum-matched контроль — отдельно.
