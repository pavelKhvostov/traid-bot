# Level-strength engine: описательный движок едет, предиктив УБИТ

Запрос пользователя: нейро-модуль, который с 2020 анализирует ВСЕ зоны/уровни в ±15k
цены BTC (мульти-TF, уровень=скопление зон+ликвидность, не одна OB/FVG), изучает реакцию
(сопротивление/пробой), РЕКУРСИВНО пересматривает мнение по прошлым уровням при появлении
новых, и выдаёт S/R в реальном времени + силу 1-10 с аргументами. TFs 1,2,4,6,8,12h,D,3D,W,M.

## Построено (research/level_engine/, 7 модулей + тесты)
- **le_zones** — причинные адаптеры канон-детекторов (OB/FVG/iFVG/RDRB + POC/VAH/VAL/
  HVN/LVN + BSL/SSL) на 10 TF; form_time=закрытие формир. бара (верхняя оценка).
- **le_cluster** — уровень = ПИК взвешенной плотности зон (скопление конфлюэнции) + ликв.,
  ±15k, кап ширины (анти-chaining).
- **le_interact** — реакции REJECT/BREAK/FLIP с t_touch vs t_resolved (=+H_react), σ=трейлинг ATR.
- **le_belief** — Beta-апостериор + причинный рекурсивный belief(L,T) (членство/прайор
  пере-выводятся под КАЖДЫЙ T, без мутируемого кэша) + сила 1-10 + механические аргументы.
- **le_engine** — причинный снапшот реального времени + JSON (путь отрисовки TV).
- **le_validate** — честный OOS-тест.
- Дизайн: workflow wf_ade4de60 (синтез-архитектура + 12-пунктовый lookahead-register +
  overfit-register). 8 тестов зелёные.

## ПРИЧИННОСТЬ доказана структурно (не декларацией)
snapshot() усекает данные до <=T -> срез детектора физически не достаёт будущего. Тест
**future-mutation**: снапшот на T БАЙТ-В-БАЙТ идентичен с будущими барами и без них.
Каждая реакция входит в belief только при T>=t_resolved. Рекурсия («зона 3 меняет мнение
по 1,2») = новый member к T входит в прайор -> перевзвешивает все прошлые реакции под T.

## ЧЕСТНЫЙ вердикт: предиктив УБИТ (как все прошлые попытки силы зоны)
le_validate: каждая реакция = тест (y=1 REJECT/удержал, y=0 BREAK/пробит), сила считается
СТРОГО ДО теста, БЕЗ survivorship (пробитые уровни в популяции).
- **BTC: AUC(strength)=0.531** (≈монетка). Конфлюэнс sumw сам = 0.539 (вся реакция/decay/
  рекурсия НЕ добавляют над сырым dwell-конфлюэнсом). has_liquidity=0.500 (мёртв).
  По годам: 0.63/0.48/0.52/0.52/0.51/0.49/**0.46** — **2025-2026 НИЖЕ монетки**, hold(8-10)
  в 2026 = 0.44 (сильные держат МЕНЬШЕ базы 64%).
- **ETH: AUC=0.545** (едва выше). **SOL: вырожденно** (n=100, base 99%).
- Бар проекта (AUC>0.55 + год-стабильно + кросс-ассет) НЕ пройден ни по одному пункту.
→ То же, что etap_251/zone_reactor/etap_192 (сила зоны ≈ 0.54, нет edge). Подтверждение:
сила уровня = переупакованный dwell-time/близость/волатильность, прогноза в ней нет.

### Density-matched null (le_validate_null, BTC) — РЕШАЮЩЕЕ
- **Наивный shuffle-y null: p=0.001** («значимо!») — ЛОВУШКА: наивный нуль ложно
  подтверждает edge, т.к. не контролирует конфаунд.
- **Стратиф-shuffle силы ВНУТРИ бинов [dist × atr]: null mean 0.530, p=0.467** — real AUC
  0.531 сидит В ЦЕНТРЕ нуля. Контролируя близость×волатильность, сила НЕ добавляет НИЧЕГО.
- **Инкремент:** AUC[y~dist+atr]=**0.630** vs [+strength]=0.573 (Δ −0.058) — близость+vol
  предсказывают удержание/пробой ЛУЧШЕ силы; сила = шумный прокси того же конфаунда.
  (dist+atr 0.630 сам = тривиальная геометрия σ-порогов, не edge.)
Вывод: предиктив убит не только год-нестабильностью, а чисто — сила = конфаунд.

## Решение
- **Predictive KILL**: `strength` НЕ прогноз. `predicts_hold=False` в JSON (вшито).
- **Descriptive SHIP**: движок едет как ОПИСАТЕЛЬНАЯ карта — где скопления зон по 10 TF,
  что реагировало исторически (REJECT/BREAK/FLIP), есть ли магнит/ликвидность, с аргументами.
  Полезно как situational awareness / разметка S/R, НЕ как сила-предсказание. Соответствует
  духу проекта («это состояние, а не прогноз»).
- Веса силы НЕ перекалибровывал под историю (это был бы оверфит, register #12).

## Оговорки
Density-matched null прогнан (см. выше) — KILL подтверждён чисто. Полные абляции
(рекурсия vs flat-sum, граф-граф) не нужны: AUC и так = конфаунд. SOL-прогон вырожден
(мало разрешимых пробоев на его масштабе) — доработать классификатор для SOL, если
понадобится. Урок (в known-pitfalls): наивный shuffle-y null ложно подтверждает edge
(p=0.001); правильный null = стратиф-shuffle внутри бинов конфаунда (близость×ATR).

## v2.1 — 2-й проход самокритики (wc5wmut5u, эмпирический factor-variance аудит)
Замерена дисперсия каждого фактора по 38 уровням → нашлись мёртвые/баги:
- **W (ширина) МЁРТВ** (std 0.029): мерил ширину кластер-полосы (упёрта в cap 0.012·price),
  не уровень. → переделан в **согласие mid'ов участников** (тесно совпадают TF = сильнее).
- **Q (реакция) МЁРТВ** (std 0.086): bestQ≤0.28 (слабые отбои) × τ=45д крушил (медиана
  возраста 181д) × gate. → τ_Q=270д, clip/2, своя свежесть (без двойного штрафа). std→0.183.
- **freshness gate САТУРИРОВАН** (floored 0.40 у 34/38). → τ 1.5→6, пол 0.5; гейтит только
  ликвидность (потребление), Q на своей recency.
- **directional collapse**: why['side'] считался и ВЫБРАСЫВАЛСЯ. → Q/K РАЗДЕЛЕНЫ на
  support(подход сверху) vs resistance(снизу); скорим сторону роли, др.сторону в аргумент.
- **+ORDER-FLOW absorption (audit #6)**: у нас был неиспользуемый CVD/delta (1h_flow.csv).
  Добавлен ОПИСАТЕЛЬНЫЙ фактор: сколько объёма поглотила лучшая реакция + знак delta
  («поглощены продажи/покупки»). **Null-тест (_val_flow_null, n=4402): AUC(absorp→hold)=0.504**
  (≈монетка, ещё хуже общего 0.531), стратиф-null p=0.404, инкремент над dist+atr Δ−0.004,
  год-нестаб (0.46-0.64). Причина: signed-delta на окнах реакций крошечный (|δ| медиана 0.05 —
  объём есть, направления нет). → **DESCRIPTIVE-ONLY, НЕ scored.** Даже signed-flow ось — монетка.
Итог: спред 3-6 → **4-7** (Q/K ожили), аргументы богаче. Тесты 8/8 (future-mutation цел).
В LIVE-снапшоте has_magnet/has_liquidity флагуются у 21/22 из 38 (has_liq=0 был артефакт ТОЛЬКО
валидации: VP form_time=T, для историч. теста Tt<T фильтруются).

**#8 HTF-якорная кластеризация — ПОПРОБОВАНА и ОТВЕРГНУТА** (cluster_htf оставлен в коде как
память): wide HTF OB-боксы chain-merge в мега-якоря → 4 уровня, один на 10k зон (хуже
density-peak 38). И ГЛУБЖЕ: C сатурирует НЕ из-за кластеризации, а из-за РЕАЛЬНОСТИ ДАННЫХ —
за 2.5г зоны навалены на каждую приценовую полосу по всем TF, любой узкий бэнд near price ловит
все TF. → C/W (структурная ось) принципиально не дифференцирует near price; дифференциация идёт
от РЕАКЦИИ/ПРОБОЯ/ORDER-FLOW (v2.1), не от структуры. density-peak оставлен. Прогноз #8 не чинил
бы в любом случае (стена). Урок: «бутылочное горлышко = кластеризация» из аудита оказалось
неверным — горлышко = плотность данных, его кластеризацией не убрать.
