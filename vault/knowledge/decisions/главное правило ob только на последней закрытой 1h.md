# Главное правило: подтверждение только на последней закрытой 1h

## Решение

Сигнал отправляется подписчикам **только если** время первого подтверждения 1h
**равно** времени последней закрытой 1h-свечи на момент диспатча.

## Где живёт в коде

[scanner.py:200-202](../../../scanner.py#L200-L202):

```python
last_1h_open = pd.to_datetime(df_1h.iloc[-1]["Open time"], utc=True)
...
if confirm_time != last_1h_open:
    continue
```

Подтверждение, найденное «в прошлом» (на любой более ранней закрытой 1h-свече),
**игнорируется** — ни broadcast, ни `mark_sent` не происходит.

## Зачем

- **Свежесть.** Без правила сразу после старта/реконнекта в Telegram прилетает пачка
  старых сигналов, которые исторически валидны, но рынок уже прошёл — это шум.
- **Идемпотентность по тиковому циклу.** WS даёт нам ровно одну закрытую 1h-свечу
  за такт; правило гарантирует, что один такт = максимум один свежий сигнал на стратегию
  × source_tf × symbol.
- **Согласованность с `_prefill_today_signals`** ([[prefill silent при старте]]) —
  prefill маркирует сегодняшние подтверждения как sent; диспатч отрабатывает только
  на новой закрывшейся 1h, где prefill ещё не был.

## Граничный случай

При `full_rescan = True` ([scanner.py:150](../../../scanner.py#L150), на закрытии 1h)
сканируются все source_tf за UTC-день, но фильтр `confirm_time == last_1h_open`
всё равно отсекает старые подтверждения. То есть полный пересканинг работает
как «найти ли подтверждение, у которого `confirm_time` совпадает с этой 1h».
