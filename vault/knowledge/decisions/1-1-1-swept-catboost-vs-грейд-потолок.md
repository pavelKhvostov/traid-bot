# 1.1.1 SWEPT: CatBoost НЕ бьёт ручной грейд; грейд = потолок (и переусложнён)

etap_251 + adversarial workflow (wf_58bb5ac0, 100+ OOS-конфигов, FWER-мультитест,
bootstrap-CI, год-стабильность). Датасет 555 дедуп-сделок BTC+ETH+SOL, fixed RR=2.2,
17 фич (структура + модуль аналитики). Цель пользователя: улучшить предыдущее (etap_249).

## CatBoost как фильтр — стена learned-meta (5-е подтверждение)
in-sample AUC **0.906** → OOS(train<2024) **0.466** (хуже монетки), permutation p=0.80.
Децили OOS перевёрнуты (топ-P → худший WR). 100+ конфигов (cutoff/depth/l2/feature-
subset/per-symbol/walk-forward) — НИ ОДИН не прошёл бар (OOS AUC>0.55 И p<0.1 И спред>13пп).
Лучшие кандидаты — шум: ETH per-symbol AUC 0.584 p=0.13 n=66; POOL num_only AUC 0.565
но спред +8.4пп p=0.12, схлопывается до +2.7пп на 2024-26. **Не внедрять.**
CatBoost-importance ПОДТВЕРЖДАЕТ фичи грейда (risk_pct 16 / eff 13.6 / session 11.6 /
gauge 10.5 / dow 10.3) — модель согласна ЧЕМ, но не может скомбинировать без переобучения.

## Ручной грейд — реальный edge, но ПОТОЛОК и ПЕРЕУСЛОЖНЁН
- edge не cherry-pick: POOL спред +13…+21пп при всех cutoff, p=0.001(ALL)/0.034(OOS);
  BTC год-стаб 7/7 (+30пп ALL). Экономически реален на BTC.
- НО: 5-компонентный композит **НЕ бьёт одиночное правило «не широкий SL»** (risk_pct<p67):
  одиночка BTC OOS +25.2пп, expR +0.65R CI[+0.26,+1.04], keep 78% — против грейда
  +18пп keep 52%. **Грейд режет объём без выигрыша в спреде.** Carrier edge = ширина SL.
- кросс-ассет: на чистом OOS≥2024 значим только BTC (p=0.075, n=85 мал); ETH p=0.31,
  SOL p=0.16 — CI пересекают 0. Стена «не переносится между активами» подтверждена.

## Ни одно доп-правило не выживает (атака 3, 17 кандидатов + dow)
dow=вторник 25% выглядит сильно, но FWER-permutation p=0.24, Bonferroni провал.
p_green/trend_hold/eff/top_tf/macro_tf — ухудшают/нейтральны OOS. htf_tf=2h «улучшает»
спред, но FWER p=0.32, разбавляет WR грейда 60→50%, год-стаб 4/6 = артефакт.
Структурное наблюдение (НЕ фильтр): htf=1h-сделки слабее 2h на всех 3 монетах
(BTC 38 vs 49%) — свойство стратегии, но как фильтр не проходит честную проверку.

## Решение
1. **CatBoost-фильтр — не внедрять** (стена). Допустим лишь как калибровочная диагностика.
2. **signal_grade оставить как есть** (контекстная подсказка, не хуже) ИЛИ для BTC свести
   к одному правилу «широкий SL (risk_pct≥p67) → слабый сетап» (проще + больше объём).
   Рекомендация: оставить дисплей грейда (информативен), но знать: carrier = ширина SL.
3. ETH/SOL — без грейда (уже гейтнуто на BTC, etap_250).
4. Дальнейший прирост 1.1.1 искать НЕ в learned-meta/фильтрах поверх этих фич (исчерпано
   5×), а в (i) сырых сигналах волн 1-2 (HAR/DVOL — в дневной модели, не в трейд-фильтре),
   (ii) исполнении/TP. Floating уже проверен = ноль (etap_241).

## Чем УЛУЧШИЛИ предыдущее (методологически, не результативно)
Прогнали грейд и CatBoost через единый честный бар (OOS + null + FWER + bootstrap +
год-стаб): доказали, что прежний грейд = потолок, а не прорыв; carrier = «узкий/широкий
SL» + BTC; кросс-ассет-значимость иллюзорна; ML не помогает. Снято ложное ожидание апгрейда.
