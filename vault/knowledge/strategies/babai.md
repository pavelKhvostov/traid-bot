# Стратегия «бабай» — дизайн (отложена)

LONG-only сетап на 12h, построен вокруг **OB с выраженной зоной ликвидности**.
Спроектирован 2026-05-19, забэктестен, отложен после слабого результата.

## Сетап (LONG only)

| Элемент | Правило |
|---|---|
| **OB** | LONG OB с явно выраженной зоной ликвидности на 12h (canon: [[что такое OB с явно выраженной зоной ликвидности]]) |
| **Зона ликвидности** | `[prev.low, cur.low]` (canon) |
| **Вход** | `entry = prev.low + 0.9·(cur.low − prev.low)` — buy-limit |
| **Стоп** | `sl = prev.low + 0.2·(cur.low − prev.low)` |
| **Зона 1** | `[cur.low, cur.high]` (диапазон cur-свечи между экстремумами) |
| **Фильтр зоны 1** | в зоне 1 должен быть LONG FVG на 2h / 3h / 4h, окно поиска FVG = окно OB (24h, `[prev.open, cur.open+12h)`), пересечение зон. i-FVG **не используется**. |
| **TP** | `bottom` LONG FVG из найденных в зоне 1; если несколько — **самый низкий bottom** |

## Исполнение

- Сетап подтверждается на закрытии `cur+1` (нужен LL-фрактал для маркера ликвидности).
- Buy-limit на `entry` активна сразу с момента подтверждения, **без срока годности**.
- Бэктест — честный limit-fill (ждёт фактического касания 1h-low). Pitfall-safe.

## Замер 2026-05-19 (BTCUSDT 12h, с 2020)

| Стадия | N |
|---|--:|
| Все LONG OB с выраженной зоной ликвидности | 58 |
| − без LONG FVG в зоне 1 | −5 |
| − вырожденные (TP ≤ entry) | −12 |
| **= валидных сетапов бабая** | **41** |

| Результат | Значение |
|---|--:|
| filled / unfilled | 37 / 4 |
| win / loss | 25 / 12 |
| WR (win / win+loss) | 67.6% |
| **Сумма R** | **+4.78R** |
| Avg win RR | 0.67 |
| R на сделку | +0.13 |
| Минусовых лет | 1 (2020: −1.67R) |

CSV: `signals/babai_backtest_12h_2020.csv` (58 строк).

## Почему отложена

- Edge тонкий: +0.13R/сделку.
- Правило «TP = самый низкий bottom» делает таргет крошечным (avg win 0.67R) и даёт 23% нерабочих сетапов.
- ASVK Trend Line и Money Hands — компоненты, названные пользователем первыми — **до сих пор без роли** в сетапе.
