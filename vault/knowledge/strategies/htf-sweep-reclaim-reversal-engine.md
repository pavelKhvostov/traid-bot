# ❌ REFUTED: HTF sweep+reclaim / MR-у-экстремумов = ENTRY-BAR LOOKAHEAD, не edge

**ВНИМАНИЕ: эта стратегия ОПРОВЕРГНУТА. Первоначальный «+0.216R/PF1.43/год 7/7» был артефактом lookahead.**
Оставлено как честный негатив + урок.

## Что было заявлено (ОШИБОЧНО)
Сравнение движков → «Дв.3 HTF sweep+reclaim (12h/1d) — победитель: +0.216R, PF 1.43, год 7/7, sym 3/3»;
specificity-контроль показал, что edge = generic MR у экстремумов (+0.58–0.78R, PF 2.6–3.9).

## Почему ЛОЖНО — entry-bar lookahead
Вход = `close[j]` (close бара j), а симуляция проверяла SL/TP начиная С БАРА j (`range(ei, end)`).
Но high/low бара j УЖЕ сформированы к моменту close-входа → «фантомные» мгновенные TP на баре входа →
завышение WR/PF. Фикс = управление со СЛЕДУЮЩЕГО бара (`range(ei+1, end)`).

## Честные числа ПОСЛЕ фикса (BTC/ETH/SOL, фьюч нетто) — ВСЕ движки перепроверены
- **MR у 12h-экстремумов: +0.601R → −0.179R** (WR 68%→36%, PF 2.9→0.73, год 7/7→0/7, null p 0.000→0.987).
- **Дв.3 sweep+reclaim: +0.216R → −0.05…−0.10R** (sym 0/3, год 2-4/7).
- **B8 range-fade: +0.04R → −0.071R** (R=0.7 лучший, sym 0/3, год 0/7; премиса «внутр.>физич.» держится но оба−).
- **Дв.1 range-fade+clear-path: +0.075R → −0.004R** (брейк-ивен; clear-path −0.004 vs магнит-против −0.140).
- **Дв.2 reversal-в-зону: −0.023R → −0.099R** (мёртв).
ВЫВОД: НИ ОДИН новый движок не даёт нетто-edge. Весь «парад» = entry-bar lookahead.

## ВАЖНО: магнит/clear-path ФИЛЬТР всё же несёт информацию
Даже на мёртвой базе Дв.1 фильтр Vadim корректно ранжирует: clear-path −0.004R vs магнит-против −0.140R
(спред +0.136R). → его зоны = реальный сигнал для ОТБОРА/сайзинга, но не создают edge из ничего — нужна
уже-положительная база. Единственная чистая положительная база сессии = arc-fade (+0.164R) → +конфлюэнс +0.309R.

## Что осталось ЧИСТЫМ (вход на OPEN следующего бара — lookahead отсутствует)
Доверять ТОЛЬКО бэктестам со входом на `open[i+1]`: arc-fade + analytics-конфлюэнс **+0.309R** (mtf+clear-path,
тонко, ETH слабее), arc **+0.164R**. Новые движки честного edge не дали. См. [[ta-curved-patterns-arcs-and-data-discovered-shapes]].

## Урок
Любой WR ≥65% / PF ≥2.5 на наивной стратегии = немедленно проверить entry-bar lookahead (вход не на open →
управление обязано идти с ei+1). См. [[known-pitfalls]]. Также: красивые null+грид НЕ спасают, если базовый
сим имеет lookahead (null тоже его имел, но strat выигрывал за счёт бага — null ловит ТАЙМИНГ, не lookahead-сим).
