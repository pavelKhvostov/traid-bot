# Оценка сигналов 1-5 + ординальная нейросеть (pavel) — etap_178

## Идея (пользователь)

Взять ВСЕ стратегии с хорошим winrate (1.1.1, 1.1.2, 1.1.3), для каждого их сигнала посчитать R-исход ПО ПРАВИЛАМ стратегии, присвоить класс качества **1-5** (1=плохой/не использовать, 5=идеал), обучить нейросеть предсказывать класс. Цель — фильтр: брать только высокие классы.

## Метка 1-5 (вокруг целевого TP=2.2R стратегий 1.1.x)

Каждый сигнал = сделка (entry=mid FVG, SL=15%·OB-depth, RR-цель 2.2). Гонка TP(2.2R) vs SL по 1m (limit-fill, НЕ instant). Класс:

- **1** = SL раньше TP (убыток) — «не использовать»
- **2** = не дошёл до 1R
- **3** = 1-2R (близко к TP)
- **4** = взял TP 2-3R (ЦЕЛЬ достигнута)
- **5** = пробил 3R+ (идеал)

Это и есть «winrate + R»: класс 4+ = взял рабочий TP.

## Данные и стандарты

948 сигналов (1.1.1=325, 1.1.2=1117, 1.1.3=252 → дедуп 980 → 948 с исходом), 6 лет BTC 1m. Фичи (86) = арсенал etap_177 (sweep/DOL, зоны, Bulkowski, SADF, frac_diff, entropy) + признаки сигнала (strategy_id, direction, risk%). Ординальная нейросеть **CORAL** (cumulative-logits, учитывает порядок 5>4>3>2>1). Стандарты López de Prado: Purged K-Fold + embargo + uniqueness weights. PyTorch/MPS.

## Защита от lookahead (known-pitfalls)

- fill-scan стартует ПОСЛЕ закрытия entry-свечи (signal_time + длительность fvg_tf) — [lookahead-15min-vs-tf_duration]. Для 1.1.3 (fvg_tf=1h/2h) корректно 60/120 мин, не хардкод 20.
- гонка по реальным 1m high/low, не instant — [etap-42-instant-fill].
- дедуп по (strategy, time, direction, entry) — [multi-shot-2.3x].
- фичи ≤ момента сигнала; tie TP/SL в одном баре → SL раньше (anti-optimism).
- **Sanity shuffle-grade: ρ = −0.055** (≈0) → lookahead'а НЕТ.

## Результаты (OOS test 2025+)

**Распределение:** grade 1 (SL) = 65%, grade 4-5 (взял TP) = 34%. **WR по TP=2.2R = 34.3%.**

**Ординальная NN:** CV ρ=0.153, TEST ρ=0.110 (слабая корреляция).

**НО монотонность работает (главное):**

| Предсказанный класс | n | реальный WR по TP (grade≥4) | mean R |
| --- | --- | --- | --- |
| pred=2 | 259 | 34% (= baseline) | 4.25 |
| pred=3 | 73 | **48%** | 4.17 |
| pred=4 | 14 | **57%** | 8.17 |

**TOP (score≥3.5): WR_TP 57% vs baseline 38%, lift ×1.52** (n=14).

## Честная оценка

1. **Как фильтр идея РАБОТАЕТ:** чем выше класс предсказала сеть, тем выше реальный winrate (34%→48%→57%) и R (4.25→8.17). Брать только сигналы класса 4-5 → winrate с 34% до 57%. Это ровно то, что просили («1 не используй, 5 идеал»).
2. **НО сигнал слабый:** ρ=0.11 на полном тесте, top-бакет всего 14 сигналов (мало для уверенности). Качество сигнала 1.1.x определяется в основном тем, что происходит ПОСЛЕ входа (рыночный шум), а не предсказуемо из контекста на входе.
3. **WR по TP=2.2R = 34% baseline** — это честный реальный winrate сигналов 1.1.x при RR=2.2 (совпадает с vault: 1.1.2 WR 44%, 1.1.3 ниже; pooled 34%).
