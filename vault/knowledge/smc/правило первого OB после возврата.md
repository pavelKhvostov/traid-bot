# Правило первого OB после возврата

> **Статус:** правило по-прежнему актуально, но теперь это **частный случай OB-1h**
> внутри [[три типа подтверждения 1h ob fvg rdrb]]. Сама функция
> `find_first_ob1h_in_zone` живёт в `ob1h_core.py`, но в live-диспатче проекта
> используется расширенная `find_first_confirmation_in_zone`, которая берёт OB-1h
> как первый из трёх возможных типов подтверждения.

## Суть

Для стратегий с возвратом в зону старшего ТФ засчитывается **первый** паттерн
подтверждения, сформировавшийся после того, как цена вернулась в зону. Любой
последующий подтверждающий паттерн в той же зоне — игнорируется (зона переходит
в `FIRED`/удаляется через `mark_sent`).

## Алгоритм (псевдокод, частный случай OB-1h)

```
zone = создана из паттерна старшего ТФ (OBx4 / OB / RDRB / FVG / молот / марубозу / снятие)
на каждой закрытой 1h-свече после zone.trigger_time:
    if cur.close < zone.bottom (LONG) or > zone.top (SHORT):
        # зона мертва, выход
        return None

    # ждём первого возврата в зону (low/high пересекают [zb, zt])
    if not вернулись: continue

    # после возврата — пара (prev, cur)
    if prev пересекает зону and (prev red, cur close > prev open) [LONG]:
        FIRE OB-1h
        return
```

## Современная версия — три типа

В live-диспатче ([scanner.py::_dispatch_strategy](../../../scanner.py)) используется
`find_first_confirmation_in_zone`, которая на каждой 1h-свече параллельно проверяет
**OB-1h, FVG-1h, RDRB-1h** и возвращает первый сработавший в приоритете
OB → FVG → RDRB. См. [[три типа подтверждения 1h ob fvg rdrb]].

Главное правило **первого** подтверждения сохраняется: после первой удачной 1h-зоны,
дальше функция не ищет.

## Почему именно первый

- **Снижает ложные срабатывания.** Цена в зоне часто качается; второй и последующий
  OB обычно срабатывают, когда trend-следящие участники уже вышли — RR деградирует.
- **Совпадает с методологией SMC.** Институциональный ордер обычно исполняется при
  первом тесте зоны.
- **Упрощает дедуп.** `mark_sent` по ключу `{strategy}|{symbol}|{source_tf}|{direction}|{confirm_iso}`
  — после fire зона больше не отстреливает.

## Что НЕ попадает под это правило

- `s4` снятие фрактала исторически считалось атомарным событием без возврата.
  В текущем коде s4 идёт через тот же `find_first_confirmation_in_zone` — фитильная
  зона работает как обычная зона возврата с подтверждением 1h.
