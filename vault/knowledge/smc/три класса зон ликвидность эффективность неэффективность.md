# Три класса зон проекта — ликвидность / неэффективность / эффективность

> **CANON.** Терминология пользователя (2026-05-20). Все зоны проекта
> делятся на три ортогональных класса. Использовать эти термины в любых
> разговорах о SMC-зонах.

## Зоны ликвидности

**FH** (Fractal High) и **FL** (Fractal Low) — фракталы Билла Уильямса.

Смысл: места скопления стопов / лимиток, куда крупный участник может прийти
за ликвидностью.

См. [[фракталы билла уильямса]].

## Зоны неэффективности

**FVG** (Fair Value Gap).

Смысл: места, где цена прошла слишком быстро и оставила гэп — рынок «не
успел» торговать в этом диапазоне, поэтому ожидается возврат и заполнение.

См. [[что такое fvg]], [[универсальные определения OB и FVG]].

## Зоны эффективности

В этот класс входят:

- **RDRB V1** — intersection-форма, узкая зона пересечения фитилей
  anchor+trigger, ограниченная телами.
- **RDRB V2** — intersection + расширение до тела anchor.
- **maxV** в свече (ViC ASVK) — «блок наторгованный»: уровень / зона
  LTF-свечи с максимальным dirVolume внутри HTF-бара. Эффективный уровень,
  где реально прошёл объём.

Смысл класса: места, где цена эффективно отреагировала или эффективно
наторговалась — реальные места активности.

См. [[что такое rdrb]] (V1 и V2 канон, V3 отклонён),
[[vic-asvk-indicator-python]] (формула maxV, LTF=15m для D-чарта).

## Замечания

- **OB** (Order Block) **не входит** в эту классификацию — отдельный
  SMC-примитив. См. [[что такое order block]].
- Классификация ортогональна направлению: каждый класс имеет LONG и SHORT
  варианты.
