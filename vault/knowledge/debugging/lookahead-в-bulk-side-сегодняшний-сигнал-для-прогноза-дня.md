# Lookahead в фиче bulk_side: сегодняшний сигнал нельзя для прогноза того же дня

## Что было
В nowcaster направления (etap_211) контекст-фича `bulk_side` (сторона последнего
Bulkowski-сигнала) для дня **d** включала сигнал, сработавший на 12h-баре **00:00/12:00
самого дня d**. Прогноз дня d делается as-of конца d-1 → сигнал, закрывшийся ВНУТРИ дня d,
это будущее.

## Симптом
Контекст-only «прогноз направления вперёд» дал **AUC 0.628** — выше доказанной стены
0.50 (etap_198). Подозрительно: один признак (Bulkowski) тянул вверх. Sanity: цель
close-to-close дала тот же 0.628 (день-цвет и close-to-close — разные цели, одинаковый
AUC = артефакт, не сигнал).

## Причина
`bulk_side = last.reindex(d.index).ffill(limit=10)` — `last` индексирован днём сигнала;
reindex на d.index включает сигнал ДНЯ d (12h-бар закрывается в течение суток d).

## Фикс
Сдвинуть на 1 день: `bulk_side = last.shift(1).reindex(...)` — для прогноза дня d брать
только сигналы ≤ d-1. После фикса: контекст-only AUC **0.628 → 0.512** (= монетка,
стена подтверждена). ML перестал бить наивный ret_k.

## Правило избегания
Любая фича-«последнее событие» (last signal/pattern/level) на дневном баре, если событие
может произойти ВНУТРИ прогнозируемого дня — **сдвигать на shift(1)** (только прошлые дни).
Подозрительно высокий AUC прогноза-вперёд (>0.55 на направлении) — ПЕРВЫМ делом искать
такой leak. Shuffle-тест полной модели его НЕ ловит (перемешивает весь target) — нужна
**проверка контекст-only части отдельно** + sanity на альтернативной цели.

[[Дневная HTF-фича на 12h-баре = lookahead]].
