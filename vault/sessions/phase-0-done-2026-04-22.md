## Решения (Decisions этой фазы)

| ID | Решение | Обоснование |
|---|---|---|
| D-01 | `AppState.update_df_with_candle` НЕ пишет в CSV; orchestrator вызывает `save_df` отдельно | SPEC §8.3 разделение ответственности, легче тестировать без side-effects |
| D-02 | `compose_from_base` + unit-тесты (9→3, 8→2) мигрируем в Phase 0 | compose критичен для Phase 1, логика должна быть покрыта до стратегий |
| D-03 | Delta-fetch при WS reconnect — в Phase 0 | Закрывает NFR-2.3, иначе пропущенные за дисконнект свечи потеряны |
| D-04 | Все 13 правок TelegramClient — в Phase 0 | Ядро bot-engineer работы, делать полностью сразу, а не дополнять потом |
| DEC-tf-lower | `tf_to_pandas_rule` возвращает lowercase `"3h"`/`"2d"` | pandas 3.0.2 не принимает uppercase (см. `vault/knowledge/decisions/pandas-frequency-lowercase.md`) |
| DEC-utcnow | `pd.Timestamp.utcnow()` deferred до Phase 4 | Depr в pandas 4, работает в 2.x/3.x; концентрированная чистка времени в backtest слое |
| DEC-dotenv | `load_dotenv(dotenv_path=Path(".env"))` явный путь без walk-up | Без этого тесты подтягивают реальный токен из project root |

1. **Channel `@ASVK_Power_Zone`: 403 Forbidden.** Бот не админ канала. Обойти нельзя без
   физического доступа к каналу. **Workaround на MVP:** доставка через ЛС (state.users).
   Когда будет доступ — добавить бота админом и переключить broadcast.

2. **`ADMIN_IDS=901107007` устарело.** Это chat_id из `reference/obxxx.py:63`, автор
   исходника. Реальный админ **886807304** (Pavel). Обновить в `.env` и `config/strategies.yaml`
   при запуске Phase 1.

3. **pandas RuntimeWarning для compose 2d:** `origin="epoch"` не применяется к D-частотам.
   2d composition работает, но выравнивание UTC-окна не гарантировано. Phase 1 добавить
   регрессионный тест на границы 2d-окон (s1 использует 2d в HTF range) и, если нужно,
   переписать compose без `origin` для D/W-частот.
