# Нейро-модуль: мета-селектор не разделяет good/bad ни на каком пуле

Продолжение нейро-ветки (etap_175-178 от 2026-06-09). Задача модуля: на момент
формирования сигнала отличать удачные сетапы от неудачных, обучить модель,
отсеять плохие. Дали задаче исчерпывающий честный шанс — закрыли вопрос.

## Что сделано (etap_179-182)

**etap_179 — geometry grid на Bulkowski (без ML).** Разделили вопрос геометрии и
селекции. 28 ячеек SL×TP (struct / atr1.5 / atr3 / cap3 × measured / half / rr1-3 /
pct5/8), unfiltered net_R на TRAIN+TEST. Итог: **0 ячеек плюсует на обоих сплитах**;
per-year показал, что PnL = направленная бета года (лонги в бычьи, шорты в 2022),
ось стопа инвертируется train↔test. Bulkowski-пул ed&apos;а не несёт. structural-стоп
(что использовали лейблы 178) — слабейшая геометрия.

**etap_180 — sequence NN (MLP) на сырых окнах.** Лейбл geometry-neutral (симметричный
first-touch ±5%, base 50.8%), purged+embargo CV. raw / ctx / combo, per-asset+pooled.
Лучший OOS AUC 0.577 (1 из 12, непоследователен) → разделения нет. Подтвердилось,
что прежний CV 0.573 был частично утечкой (после purge → 0.51).

**etap_181 — ПОЛНЫЙ набор фич проекта (~250 из etap_170)** на Bulkowski-сигналах:
зоны OB/FVG×5TF + сила, multi-TF Hull, sweep, Lopez-микроструктура, USDT.D, blocks.
HGB+MLP. resub 1.0/0.98, purged-CV 0.56/0.59, **OOS 0.52/0.52**. Даже весь alpha
проекта не разделяет. per-pattern: hs_bottom/v_top/triple_top 0.62-0.66, но n=25-36
(шум, multiple testing). Отсев по P: +1.2пп ценой −2пп при росте τ.

**etap_182 — мета-селектор на i-RDRB+FVG (пул с ДОКАЗАННЫМ edge).** Канон smc-lib
(i-RDRB C1-C4 + FVG C3-C4-C5, entry=mid block, SL=pattern low/high, TP=RR1), исход
на 1h, BTC+ETH+SOL.
- **Baseline подтвердил robust OOS edge**: BTC WR 63.3%/+218R, ETH 64.6%/+225R,
  SOL 63.2%/+202R; OOS WR 62.6%, +257R, плюс каждый год (2024:+95 / 2025:+108 / 2026:+32).
- **Мета-селектор не разделяет**: HGB resub 1.0 / purged-CV **0.47** / OOS 0.544;
  MLP 0.54/0.46/0.50. Отсев по P: WR +0.9…+1.5пп, но ΣR падает (+257→+195).
- Топ-importance = `R_pct`/`R_over_atr` — геометрия R/ATR (известный фильтр), не контекст.

**etap_183 — мета-SIZING на i-RDRB+FVG (предсказать магнитуду, не win/loss).** Гипотеза:
бинарный исход непредсказуем, но MFE-магнитуда — частично да → селективно расширять TP
на runner'ах. Цель y_big = MFE≥2R.
- runner-разделимость: HGB resub 1.0 / purged-CV 0.512 / **OOS 0.515**; MLP 0.49/0.50. Монетка.
- payoff OOS: baseline fixed-RR1 +257R; blanket RR2 +213, RR3 +155 (расширение ВРЕДИТ —
  подтверждает [[floating-tp-only-helps-low-wr-strategies]]). Селективная политика
  RR2/RR3@runner — **каждая комбинация ≤ baseline** (Δ −6…−90); лучшая RR2@τ0.6 +251
  (Δ−6), по годам ничья. **Не бьёт fixed RR1.**
- Магнитуда так же непредсказуема, как бинарный исход.

## Главный вывод

**Обучаемый мета-селектор не даёт надёжного разделения good/bad НИ В КАКОЙ постановке
— ни фильтр (бинарный win/loss), ни sizing (магнитуда MFE) — ни на пуле без edge
(Bulkowski=бета), ни на пуле с сильным edge (i-RDRB+FVG).**
У SMC-сетапов исход не имеет отделимой пред-входной сигнатуры сверх геометрии сетапа.
Edge живёт в определении сетапа и R/ATR, а не в контексте, выучиваемом сетью. Лучший
«фильтр» — простое геометрическое правило R/ATR, ML его не бьёт. Воспроизвели
закон [[vadim 12 confluens asvk]]: confluence/фильтр повышает R/tr, режет ΣR.

Связь с [[López de Prado]] (изучен в этой сессии): нет теории, почему сигнал
предсказывает свой исход → нечего выучивать; resub 1.0 → CV ~0.5 → OOS ~0.5 =
учебный train-overfit (§1.4.1); KFold без purge тёк (AFML гл.7).

**etap_184 — изучение каждого Bulkowski-паттерна В ОТДЕЛЬНОСТИ (good=+5%>стоп).**
Пул (178-181) скрывал паттерн-специфику. Раздельно: у большинства паттернов сильные
in-sample сепараторы ИНВЕРТИРУЮТСЯ на OOS (hs_bottom rr_at_target tr0.79→te0.38;
triple_top dist_swing_lo tr0.80→te0.41; hs_top/db_eve_eve/v_bottom тоже). **Реальное
исключение — big_m** (единственный short с массой, n_test=115): два независимых
сепаратора держатся train↔test — `dist_swing_hi_pct` (tr0.64→te0.64) и `rr_at_target`
sign− (tr0.63→te0.66), mvOOS 0.593. OOS-правило: big_m при dist_swing_hi_pct≥7.35%
→ good% 68 (n=63) vs 60 (n=52). Скромный +8пп, осмысленно (много места до swing high
+ далёкий стоп = реже бустится). v_top формально «ДА», но шум (n=19/15). CSV
`output/etap_184_per_pattern.csv`.

**etap_185 — валидация big_m.** Сепаратор РЕАЛЕН: AUC держится train→te кросс-актив
(BTC 0.65→0.66, ETH 0.64→0.70; SOL слабее), good% OOS 64→69. **НО в PnL НЕ
конвертируется**: R/tr +0.070→+0.004, ΣnetR +8.1→+0.2, 2 плохих года из 3. Причина:
разделяющая фича `rr_at_target` низкий = далёкий стоп → выше hit%, но меньше R на

у big_m отделим по good%, но фильтр не торгуем; у остальных паттернов не отделим.
**Нейро-модуль на Bulkowski-паттернах закрыт полностью.**

**etap_186-188 — кластеры → своя геометрия классу (идея пользователя «делить на
классы, каждому свой стоп/тейк»).** etap_186 (KMeans K=6) per-class геометрия OOS

**etap_189-191 — МАКСИМАЛЬНАЯ попытка (свободные руки): весь универсум 12h-баров
(BTC+ETH+SOL, оба направления, ~25k образцов) × ~50 TA-индикаторов (наши + классика:

- etap_190 диагностика: НЕ чистая тавтология (close_pos один = 0.50, market-only = 0.61,

- etap_191 ФИКС: `usdtd_1d_ret` был **lookahead** (дневной возврат за дату 12h-бара =

**etap_192 — сила зоны прихода (идея пользователя «куда пришла свеча, в какую зону»).**

Причина: zone-strength предсказывает «сформируется ли пивот» (etap_167 imp 0.36 — ДРУГАЯ

**Нейро-модуль на Bulkowski закрыт во ВСЕХ постановках** (outcome-prediction /
busted-фильтр / кластерная геометрия / весь-универсум+макс-индикаторы / сила-зоны+индикаторы)
— все плюсы артефакты (бета / степени свободы / lookahead).
Дальше = p-hacking. Возврат только с новой гипотезой об источнике предсказуемости
(regime-model / exogenous-данные), не вариация модели/фич/кластеров.

**etap_193 — forensics ПУТИ + in-process предсказание (идея пользователя «читать
движение по ходу, не прогноз в t=0»). ПЕРВЫЙ РЕАЛЬНЫЙ СИГНАЛ.**
Путь каждой сделки на 1h (ранний MAE/MFE, чекпоинты +1/2/3%, снимки 6/12/24/48ч).
- Forensics: победитель держится (MAE 12h 0.13 / 24h 0.16), busted продавливает
  (0.22 / 0.39). Монотонно: MAE12 мелкий→success 74%, глубокий→39% (спред 34пп).
- IN-PROCESS модель (открытые на +12ч, путь ≤12ч → финал): **purged-CV AUC 0.599,
  OOS 0.580** (vs t=0 ~0.52) — LEAK-FREE, согласовано. Доминанта = mae12.
- mean net_R МОНОТОНЕН по квартилям P: Q1 −0.112 → Q4 +0.091 (success 51→70%).
- НО бинарный early-cut НЕ конвертирует в ΣR (Δ−3…−16): при rr~3.9 зарезанные
  победители (в Q1 их 51%) перевешивают; выход на 12ч даёт плохую цену.
- ВЫВОД: путь несёт реальную информацию об исходе (валидировано). Правильное действие
  — не cut, а СAЙЗИНГ (тилт к высоким-P). Метод (in-process management) применять поверх
  стратегии с edge (i-RDRB+FVG +257R), не на слабом Bulkowski-пуле (baseline ~+11R).
CSV etap_193_paths.csv. Это переформулировка задачи «читай путь по ходу», а не
«предскажи на входе» — единственная, что дала сигнал.

2024-2026). Это самостоятельная хорошая новость; число 2357 filled сетапов на

## Открытые опции (если возвращаться к нейро)

Все ML-постановки (фильтр + sizing) исчерпаны и закрыты. Остаётся:
1. Single-feature R/ATR-правило для i-RDRB+FVG как простой НЕ-ML фильтр (без сети).
2. Торговать i-RDRB+FVG как есть, fixed RR1 (OOS +257R / 3 актива) — это уже хороший

# Продолжение (поздняя сессия 2026-06-10): zone_reactor + чтение 3 книг

## zone_reactor — нейро-модуль «силы зоны реакции» (research/zone_reactor/)

Запрос пользователя: проект, который по 8h/12h/D/3d показывает сильные зоны (реакция
≥5%), самообучаемый, на GPU (RTX 4070 Ti Super). Построен честно, валидирован → edge нет.

**Постановка корректная (без фрактал-утечки):** единица = 12h confirmed fractal / zone-touch;
предсказываем УСЛОВНО (среди фракталов/касаний) реакцию ≥5%. ВАЖНО про etap_170 AUC 0.93:
это была **фрактал-ДЕТЕКЦИЯ-утечка** (sweep+close_pos тривиально = «это локальный low»), не
предсказание реакции.

**v1 (features.py/baseline.py/diagnostic.py):** baseline OOS AUC 0.640 выглядел сигналом, НО
diagnostic: `atr_pct` один = 0.613 (почти весь AUC), **зоны-только = 0.485 (ниже монетки)**,
per-asset зоны+ICT ~0.5. → 0.64 = ВОЛАТИЛЬНОСТЬ («хватит ли на фикс 5%»; SOL реагирует 82% vs
BTC 61%), НЕ multi-TF сила зоны. Третье подтверждение (etap_192 то же).

**zone_touch (held/broke) — ПРАВИЛЬНАЯ постановка пользователя** (не «какой фрактал даст
реакцию», а «зона на касании удержится/пробьётся»): OB+FVG на 8h/12h/D/3d → первое касание →
held(+5% раньше пробоя)/broke. 14796 касаний. После 2 фиксов утечек (intrabar SL-first вместо
close-based; фичи на last-closed баре): OOS AUC 0.728/без-width 0.679; net_R baseline вслепую
−0.013, фильтр P≥0.6 +0.586. **НО multi-TF confluence сам = AUC 0.517 (монетка)** — драйверы
RSI/premium-discount/свежесть/ширина/волатильность, НЕ confluence.

**Критика пользователя про ШИРИНУ (верная, улучшила модель):** топ-P зоны были 10-20%
шириной = мусор, геймящий лейбл (реакция 5% << ширины тривиальна). Фикс = кап ширины ≤3%.
На узких: AUC 0.671, P≥0.6 R/tr +1.396, плюс все годы. zone_width_sanity.py.

**1h-бэктест (backtest_1h.py) разоблачил 12h-оптимизм:** на честном 1h-исполнении фильтр P≥0.6
R/tr +0.10 (2025 ≈0). Причина: 1h SL-first ловит внутридневные выбивания фитилём → held% 67%→41%.

**Критика пользователя «1h≡12h» (математически верная):** 12h.low=min(1h.low) → 1h не может
дать больше стопов. Разница held% была от РАЗНОЙ логики скриптов. Собрал **чистую модель
пользователя** (backtest_user_model.py: вход=CLOSE 12h-свечи касания, единый резолв): held12==held1
у 99.5% → юзер прав. И тут **edge ИСЧЕЗ: OOS AUC 0.543, net_R −0.17, оба года минус.** Прошлые
+0.78/+1.4 были артефактом конструкции бэктеста (вход по прокс.краю + рассинхрон касание→12h-граница).

**v2 acceptance (v2_acceptance.py):** твоя идея «проторговка свечей в зоне» (volume-at-price). На
чистой модели: BASE 0.599 → +ACCEPTANCE 0.615 (Δ+0.016, реальный но малый) → +multi-TF-стек
0.603 (стек = шум). acc_vol_frac имеет importance. Единственный плюс за ветку, но мал.

**v2 self-target (v2_selftarget.py):** система сама подбирает цель (ATR-нормир. TM/SM/H) на train
через purged-CV (анти p-hacking, López §1.4.2), тест 1 раз. Выбрала TM3/SM1, CV +0.317 in-sample
→ **OOS AUC 0.548 (монетка), net_R +0.012, n=45.** In-sample цель не генерализуется.

**ВЫВОД zone_reactor: робастного торгуемого edge НЕТ** (8-й артефакт за сессию). Под чистым
исполнением зоны не разделяются. Сила зоны = «где смотреть», не «сколько пройдёт». README +

## Чтение литературы (пользователь принёс 3 книги) → СИНТЕЗ

**Dalton «Markets in Profile»** (Market Profile / Auction Theory): не прогноз, а чтение
time-price-volume в настоящем. Новое для нас — **volume-at-price**: Value Area (70% объёма), POC
(магнит), HVN/LVN (цена залипает на HVN, проходит сквозь LVN), **excess** (низкообъёмный фитиль =
разворот). Ровно жила, где у нас дрогнул сигнал (acceptance).

**Harris «Trading and Exchanges»** (микроструктура): главный тезис «uninformed traders lose
simply because they trade» (та же стена). Кто двигает цену: informed (делают цену информативной),
order anticipators (паразиты), liquidity suppliers. **~90% инструментов требуют order-book/tick с
buy/sell разметкой — у нас только OHLCV.** НО Binance в klines отдаёт `taker buy base/quote volume`
(прокси signed order flow / CVD), а наш пайплайн его ДРОПАЕТ. → можно дозагрузить и построить
CVD/delta/order-flow-imbalance.

**Grimes «Art and Science of TA»** (tested-edge): (1) база = random-walk, edge только если бьёт
случайность (= наш null-тест); (2) **edge ограничен горизонтом И магнитудой** (наш фикс +5%/10д
может быть неверным размером); (3) **«Four Trades» Вайкоффа: pullback trend-continuation =
quintessential pattern.** КЛЮЧЕВОЕ ПРОЗРЕНИЕ: всё, что мы гоняли (zone-reaction, held/broke,
reversal) — это **counter-trend mean-reversion, самая трудная низко-edge сторона.** Grimes + вся
литература: надёжный edge в **trend-continuation pullback**, который мы НИ РАЗУ не тестировали.

**Три книги сошлись:** edge НЕ в предсказании разворота из геометрии, а в **(сторона)
trend-continuation + (фичи) микроструктура объёма/потока** (volume-profile Dalton + signed order
flow Harris).

## ОТКРЫТАЯ ГИПОТЕЗА для следующей сессии (первый по-настоящему новый угол)

Тест **trend-continuation pullback** (а не reversal) + новые фичи:
- Сетап: в подтверждённом HTF-тренде — откат к зоне/POC, вход ПО тренду (сторона #1 Grimes).
- Фичи: volume-profile (POC/VA/HVN-LVN, Dalton) + signed order flow (taker-buy CVD/delta, Harris —
  дозагрузить из Binance klines) + indicators.
