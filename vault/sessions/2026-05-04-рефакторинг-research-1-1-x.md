### Фаза 3 — git mv (~5 минут)

56 коммитов на main:

- 8 README × {root, 1_1_1, 1_1_2, 1_1_3, 1_1_4, 1_2_0, rdrb, vic}

Каждый перемещённый файл получил sys.path injection в начале:

```python
# --- repo-root injection (Phase 3 refactor) ---
import sys as _sys
from pathlib import Path as _Path
_ROOT = _Path(__file__).resolve()
while not (_ROOT / "data_manager.py").exists():
    if _ROOT.parent == _ROOT:
        raise RuntimeError("repo root not found")
    _ROOT = _ROOT.parent
if str(_ROOT) not in _sys.path:
    _sys.path.insert(0, str(_ROOT))
# --- end repo-root injection ---
```

Робастный (находит root по `data_manager.py`, не зависит от глубины).
В 5 cross-import файлах 1.1.1 — дополнительно добавлен путь
`research/1_1_1/backtest/` в sys.path.

py_compile прошёл на всех 49 файлах. pytest 39/39 OK.

  можно слить с argparse `--mode={final,sensitivity}`.

аккуратного слияния. **Не приоритет**, для navigation структура папок
уже достаточна.

### Cutoff в Phase 4

`DAYS_BACK=1095` отсчитывается от `today()` — не воспроизводимо при
разных датах прогона. Решение: прогон в тот же день. Если в будущем
понадобится диagnostic refactor — захардкодить
`cutoff = pd.Timestamp("2023-05-08T00:00Z")` в скриптах. Pavel
подтвердил: «прогон в тот же день» — выбираем такой подход.
