# Живой Telegram-дашборд: BTC/ETH/SOL + TOTAL/TOTALES, кнопки, Справка

Продолжение [[2026-06-11-direction-module-honest-regime-daytype-layer]]. Из day-type
движка собран **живой Telegram-продукт**: дашборды по нажатию, авто-пуш при смене режима,
рыночный обзор. Честный посыл сохранён: «состояние дня, не прогноз».

## Что построено (etap_223–230)
- **etap_225** — дашборд дня (3 панели: цена+утр.коридор+ФОН-режим / шанс роста по часам+мёртвая зона / сколько дневного хода пройдено). Движок обучен на BTC, применяется к любой монете.
- **etap_226** — контраст тренд vs ротация (движок РАЗЛИЧАЕТ дни).
- **etap_227** — авто-пуш бот: раз в час (schtasks `LiveDashboard`) пересобирает BTC/ETH/SOL живьём с Binance, **кэширует**, шлёт **при СМЕНЕ** режима/сигнала. Подпись: режим+сигнал+шанс+«что делать»+gauge человеческим языком.
- **etap_228** — бот КНОПОК (long-poll), **постоянная нижняя reply-клавиатура**: `[BTC][ETH][SOL] [🌐 ТОТАЛ][ℹ️ Справка]`. Отдаёт последний часовой кэш (нажал 16:48 → версия 16:00). Справка = как читать сигнал/картинку простым языком.
- **etap_229** — карта статистики TOTAL/TOTALES (CoinGecko `/global` + Fear&Greed `alternative.me`): TOTAL, TOTALES(=TOTAL×(1−доля стейблов)), BTC.D, ETH.D, TOTAL2/TOTAL3, объём, F&G + авто-«чтение».
- **etap_230** — СВЕЧНЫЕ дашборды TOTAL/TOTALES из **настоящего TradingView** (`CRYPTOCAP:TOTAL`/`TOTALES` через CLI tradingview-mcp, пересемпл 15m→1h, график возвращается на BTC). Запасной источник — market-cap **basket** из топ-монет Binance (если TV закрыт).

## Архитектура / данные
- Монеты — **Binance live** (D.fetch), почасовой кэш, без дёрганья графика.
- TOTAL/TOTALES — **TV по нажатию кнопки** (не каждый час: иначе график мерцал бы постоянно). TV-CLI кратко меняет символ и возвращает на BTC; нет TV → basket.
- Статистика — CoinGecko + Fear&Greed (бесплатно, без ключа). История TOTAL у CoinGecko = только PRO (401) → потому TV/basket.
- Бот: **@new_edge_neiro_bot** (`DASHBOARD_BOT_TOKEN` в .env), `DASHBOARD_CHAT_ID=901107007`. Кэш в `state/live_dashboard/`.
