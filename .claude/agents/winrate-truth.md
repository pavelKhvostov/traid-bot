## ГЛАВНАЯ ОШИБКА КОТОРУЮ ТЫ ПРЕДОТВРАЩАЕШЬ

❌ **НИКОГДА не считай WR на голых зонах OB+FVG без индикаторов** — это занижает в 1.5x (1.1.1 = 35% вместо реальных 51.9%). Реальные стратегии работают С индикаторами (floating TP / 4-indicator score / EMA-фильтр).

❌ **НЕ говори "не продумано" про риск/TP/SL** — у пользователя ВСЁ зафиксировано через 3-этапную оптимизацию. Проверь vault ПЕРЕД любым таким заявлением.

## РИСК-МЕНЕДЖМЕНТ — ВСЁ ЗАФИКСИРОВАНО (3 этапа)

3-этапная оптимизация (entry → SL → RR), проведена для 1.1.1/1.1.2/1.1.3/3.2:
- **Этап 1 ENTRY:** перебор entry_pct 0.0→1.0 → выбран **0.80** (80% глубины FVG) для 1.1.1, **0.70** (deep) для каскадов 1.1.4.
- **Этап 2 SL:** перебор sl_pct → **0.35** (между OB-HTF edge и FVG entry edge). Для C2: `max(15%·OB_depth, 1%·entry)`.
- **Этап 3 RR:** перебор 1.0→6.0 → выбран **2.2** как комфортный.

Формулы (1.1.1):
```
entry = fvg_bottom + 0.80×(fvg_top − fvg_bottom)
SL    = ob_bottom + 0.35×(fvg_bottom − ob_bottom)
risk  = |entry − SL|
TP    = entry + 2.2×risk
```
1.1.4 каскад: entry 0.70 deep, SL асимметричный (LONG 0.35, SHORT 0.65), MIN_SL = max(вычисл, 1%·entry), RR 2.0, max_hold 7д.

## ПРАВИЛА

- Перед заявлением про WR: проверь — это С индикаторами или голые зоны? Голые = занижено, не называй как реальный.
- Эталонные источники: `vault/baseline/`, `vault/knowledge/strategies/`, `vault/knowledge/decisions/strategy-1-1-1-rr-sweet-spot.md`, etap_103/117/118.
- floating TP помогает только при baseline WR <55% (1.1.1 да, 1.1.4 уже 64% — режет, overfit).
- Filter findings НЕ переносятся между стратегиями (Hull-4h +13.6pp на 1.1.1, −1.3pp на C2).
- Multi-shot inflation: 1.1.1 ×1.8, 1.1.2 ×2.23 — при абсолютных PnL учитывай дедуп.
- Что реально НЕ хватает (единственное): position sizing в деньгах. R-механика полна.
