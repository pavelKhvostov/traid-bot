markdown# Trading Signals Bot

## Что это

Telegram-бот, который параллельно прогоняет 4 алгоритмические стратегии по
криптовалютам на Binance Spot и шлёт сигналы подписчикам в ЛС.

## Стратегии

1. **OBX4** — паттерн из 4 свечей с чередованием + FVG на 5-й.
   ТФ: 1h, 2h, 3h, 4h, 6h, 8h, 12h, 1d, 2d, 3d.
2. **FVG 4H→1H** — Fair Value Gap на 4H, валидируется, потом первый 1H FVG
   в окне 8 часов после подтверждения. Long и Short.
3. **Fractal/Vasya/FVG** — 4H фрактал low → breaker → "Вася" (фрактал на 1h
   в диапазоне breaker-свечи) → свеча подтверждения → FVG на 15m. Только Long.
4. **RDRB** — Red Day Reversal Bar, паттерн из 3 свечей. Long и Short.

## Символы

BTCUSDT, ETHUSDT, SOLUSDT

## Источник данных

Binance Spot REST + WebSocket (публичное API, ключи не нужны).

## Ключевые решения

- Все времена в UTC (как отдаёт Binance).
- Обрабатываем ТОЛЬКО закрытые свечи (`k["x"] == True` в WebSocket).
- Составные ТФ (3h, 2d) пересобираются из базовых (1h, 1d) через pandas.resample
  с `origin='epoch'`, выравнивание по UTC 00:00.
- Дедуп сигналов через `state/sent_signals.json` — ключ
  `{strategy}|{symbol}|{tf}|{direction}|{confirm_time_iso}`.
- Подписчики — список chat_id в `state/users.json`.
- На старте бота: bootstrap истории с 2022-01-01, разовый бэктест-прогон всех
  стратегий, присылает админу сводку ("OBX4 нашла N, FVG — M..."). Потом лайв.

## Формат сигнала в Telegram

⚡ OBX4  
Актив: BTCUSDT
Таймфрейм:4h
Теги стратегий: OBX4 — ⚡, FVG — 🎯, Fractal — 🔱, RDRB — ⚪.

## Стиль кода

- Python 3.13, pandas 2.2.3, requests, websockets, python-dotenv.
- Никаких ORM, базы данных, Docker — всё в CSV и JSON.
- Простые функции и dataclass-ы, никаких классов-фабрик и DI-контейнеров.
- Логи — обычный `print()`, формат `[TAG] сообщение`.
- Русские комментарии допустимы.

## Команды Telegram-бота

- `/start` — подписаться на сигналы.
- `/stop` — отписаться.
- `/status` — подписан/не подписан.
- `/lastsignal` — последний сигнал.

## Чего НЕ делать

- Не делать веб-интерфейс, дашборды.
- Не добавлять async-ORM, Redis, Celery.
- Не писать бэктест-фреймворк — только одноразовый прогон.
- Не покрывать тестами — проект одноразовый, проверка = живые сигналы.
- Не использовать TA-Lib или сторонние индикаторы — вся логика уже в
  существующем коде пользователя.

## Источники логики стратегий

В проекте лежат 4 готовых скрипта с реализацией каждой стратегии. Их нужно
ВЫТАЩИТЬ из монолитных файлов и ПЕРЕНЕСТИ в модульную структуру выше,
сохранив математику один-в-один. Не оптимизировать, не переписывать — только
аккуратно разложить по файлам.
