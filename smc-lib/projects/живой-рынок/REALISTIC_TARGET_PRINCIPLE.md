# Принцип «Реалистичной цели» (Realistic Target)

Канонический документ для проекта «живой-рынок». Корректируется по мере новых мыслей пользователя.
Claude при триггере **«принцип реалистичной цели»** / **«realistic target»** выводит этот файл целиком.

---

## 🎯 Базовая мысль

Модель должна учиться предсказывать **достижимые трейдером значения**, а не **физические max/min** свечи.

Между «физическим high дня» и «реалистично достижимым high» — есть **дельта недоступности** из-за market chaos: цена ушла выше, но удержать сделку до туда было невозможно (или нерационально).

> Удержание до физического max = удержание против собственного риск-менеджмента.

---

## 🧨 Конкретный пример

**2026-06-14**, anchor $64,500:
- Физический HIGH 24h = **$67,000** (+3.92%)
- Модель B8.1 предсказала HIGH = **$65,704** (+1.90%)

«Ошибка модели» $1,296 — НО:
- Реальный трейдер закрыл бы LONG в зоне +1.9% (RR 2:1)
- Удержание до +3.92% требует «железной воли» в хаосе
- Вероятность разворота растёт с каждым процентом без отката

→ Модель дала **достижимый realistic TP**, не «теоретический max». Это не bug, это **conservative bias**, которое **золото для трейдинга**.

---

## ❌ Что сейчас не так в наших targets

```python
target_high = log(max(price 24h) / open)   ← физический max
target_low  = log(min(price 24h) / open)   ← физический min
```

**Проблема**: модель учится предсказывать **outlier excursions** которые трейдер не может или не хочет удерживать.

Результат: либо overoptimistic TP (если модель угадает), либо мнимая «недоучка» (как в B8.1 кейсе).

---

## ✅ Что должно быть

```python
target_realistic_tp = log(first_significant_peak_after_low) / open)
target_realistic_sl = log(first_significant_bottom_after_high) / open)
```

Где **«significant peak»** = пик после которого был **разумный pullback** (например > 0.5% или > N bars).

То есть:
- Не «теоретический max», а **«первый peak от которого цена откатилась»**
- Не «теоретический min», а **«первый bottom после которого был bounce»**

Это **trading reality** target, не **physical reality** target.

---

## 🛠 Реализация — algorithm для realistic targets

```
для каждого 24h окна:
   1. Найти ВСЕ swing peaks и bottoms (Williams fractals на 1m)
   2. Отфильтровать «значимые» (с pullback'ом ≥ K%)
   3. Realistic HIGH = первый significant peak после anchor
   4. Realistic LOW = первый significant bottom после anchor
   5. Альтернатива: брать peak/bottom где RR ≥ 1:2 от ближайшего противоположного
```

Параметры (нужно настраивать):
- `K%` — минимальный pullback для «significance» (предложение: 0.5%)
- `Min RR` — минимальное соотношение для realistic TP (предложение: 1:2)
- `Max bars to first peak` — горизонт для первого pick (предложение: 12 bars на этом TF)

---

## 🎓 Философия

> **Точность важнее экстремума.**
> 
> Модель которая угадывает **realistic TP** = модель которая делает **деньги**.
> Модель которая угадывает **теоретический max** = модель которая иногда красивая в test'е, но в продакшен убыточна.

---

## 🔗 Связь с другими принципами

- **[Fresh Look]** — если цена прошла дальше realistic TP но потом откатилась, это говорит «текущая гипотеза устарела» → reset
- **[DIRECTIONS]** — для направления #2 (интрадей с плечами) realistic TP **критичен**: попадаешь часто — зарабатываешь стабильно
