# Принцип «Свежего взгляда» (Fresh Look)

Канонический документ для проекта «живой-рынок». Корректируется по мере новых мыслей пользователя.
Claude при триггере **«принцип свежего взгляда»** / **«fresh look»** выводит этот файл целиком.

---

## 🎯 Базовая мысль

После события **A** обычно следует **B**, после **B** → **C**. Если модель выучила эту цепочку, ей **не нужен close** для прогноза — она может **chain-reasoning'ом** строить план на будущее, основываясь только на текущем контексте.

**НО**: рынок живой. Цепочка может сломаться в любой момент. План — это не аксиома, а **гипотеза**.

---

## ✦ Концепт: план = гипотеза, не догма

1. На основе текущего контекста (**A**) формируется гипотеза «следующее = **B**»
2. Цена движется
3. **Если идёт по гипотезе** → продолжаем смотреть на **C**, разворачиваем план дальше
4. **Если идёт ПРОТИВ гипотезы** → ставим **отметку ✗** (crosshair)
5. План аннулируется. Формируется **свежий взгляд** на текущую ситуацию

> **Ключевое**: если ошибся в **первой** зоне — нет смысла придерживаться остальной части плана. Цепочка строится от первой проверенной точки. Если первая не подтвердилась → всё дальше неверно.

---

## ✗ Когда ставим crosshair (отметка ошибки)

- Цена **пробила** ожидаемую support/resistance в неожиданную сторону
- Цена **ушла к другой зоне**, которая не была в исходной гипотезе
- Сила движения **существенно отличается** от ожидаемой (например, ожидали отскок, получили слом)
- Time-to-event прошёл **значимо** мимо ожидаемого временного окна

---

## 🔄 Механика свежего взгляда

Когда ✗ зафиксирован:

1. **Стереть** всю предыдущую планную цепочку (B, C, D, ...)
2. **Зафиксировать новое состояние**: где сейчас цена, какие зоны активны, что произошло
3. Считать ситуацию как **новый исходный контекст** (=**A'**)
4. Сгенерировать новую гипотезу **с нуля**, забыв предыдущие предположения

---

## 🛠 Реализация в ML

| Компонент | Что делает |
|---|---|
| **Streaming prediction** | Модель держит прогноз «in flight» с метками времени и confidence |
| **Divergence detector** | Наблюдает за расхождением `actual vs predicted` в реальном времени |
| **X-mark trigger** | Срабатывает при достаточном расхождении (порог настраиваемый) |
| **State reset + re-encode** | Новая инференция с нуля, новый контекст A' |
| **Confidence decay** | Confidence старого прогноза падает по мере divergence (не бинарно ON/OFF) |
| **Divergence history** | Хранится для последующих обновлений модели |

## 💡 Философия

> «Ошибаются ВСЕ. Удерживание неправильного плана = убытки. Признать ошибку быстро = выжить.»

Это **противоположность «убеждения»**. Модель ДОЛЖНА сомневаться в собственных прогнозах при наличии противоречащих данных. Уверенность — это лучше когда подтверждается. Сомнение — это лучше когда план не сработал.

## ❓ Открытые вопросы (для развития)

- [ ] Как формализовать «значимое расхождение» — какой порог для ✗?
- [ ] Сколько X-marks подряд = регим смены характера рынка?
- [ ] Можно ли использовать **divergence как фичу** для следующих прогнозов?
- [ ] **Hierarchical X-marks**: ошибка на 4h vs ошибка на 15m — разные веса
- [ ] **Persistence**: каков «период полураспада» гипотезы?
- [ ] **Bayesian update vs hard reset** — обновлять плавно или резко сбрасывать?
- [ ] Различение «временный шум» vs «структурный break» — оба порождают ✗, но требуют разной реакции
- [ ] Накопление X-marks как сигнал для **повышения tracking error tolerance** (модель «остыла»)
- [ ] Можно ли тренировать модель **дважды**: на прогноз + на детекцию своей ошибки?
