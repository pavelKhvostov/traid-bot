# Пример анализа рынка

**Path:** `~/smc-lib/projects/пример-анализа-рынка/`
**Started:** 2026-06-14
**Goal:** Зафиксировать реальный процесс анализа Vadim'a на конкретных рыночных ситуациях. Извлечь из этого workflow canon — последовательность шагов, какие фичи смотрятся в каком порядке, какие confluence требуются для торгового решения, что разрывает thesis.

## Зачем

Все ML-проекты строятся на гипотезе «модель угадает то, что трейдер увидит». Чтобы эта гипотеза работала:
- ML должна иметь features покрывающие то, что человек смотрит
- Features должны измерять что-то осмысленное для трейдинга, а не сырую геометрию

Этот проект — **источник правды** о том, что Vadim реально анализирует.

## Структура

```
пример-анализа-рынка/
├── README.md                    — этот файл
├── workflow-canon.md            — извлечённый workflow Vadim'a (постепенно дополняется)
├── sessions/                    — лог каждой совместной сессии
│   └── YYYY-MM-DD-session-NN.md
├── snapshots/                   — скриншоты TV из сессий
└── learnings/                   — извлечённые принципы / heuristics
    └── YYYY-MM-DD-learnings.md
```

## Процесс сессии

1. Vadim ведёт анализ (открывает TV, описывает что видит и о чём думает)
2. Claude записывает дословно в `sessions/...md`:
   - Что Vadim смотрит (TF, элементы, уровни)
   - В каком порядке
   - Какие выводы делает на каждом шаге
   - Что вызывает сомнения, какие contra-сигналы он отбрасывает / учитывает
3. Claude задаёт уточняющие вопросы, когда что-то неочевидно
4. После сессии — извлечение принципов в `learnings/...md`
5. Если принципы стабильны через несколько сессий — обновляется `workflow-canon.md`

## Правила игры

- **Claude слушает, не забегает вперёд** — не предлагает решений, не делает выводов до того как Vadim их озвучил
- **Claude задаёт вопросы только когда не понимает** — не для проверки, а для уточнения
- **Vadim не объясняет всё подряд** — описывает естественно, как обычно делает анализ
- **Claude вытаскивает паттерны после** — структурирует не во время, а в `learnings/`
