# Fractal (Williams)

Локальный swing-экстремум по `(2N+1)`-bar окну: центральная свеча со строго максимальным `high` (FH) или строго минимальным `low` (FL) среди `N` соседей слева и `N` соседей справа.

Canon — **Williams Bill (BW) 5-bar fractal** с `N=2` (как в проекте по умолчанию). Параметр `N` — настраиваемый для совместимости с другими определениями (Pine WICK.ED поддерживает N=1 и N=2; LuxAlgo Liquidity Swings использует N=14 для своих внутренних целей — см. примечание ниже).

> **Класс зоны**: **liquidity** (см. [[zone-class-liquidity-inefficiency-efficiency]]). За FH накапливаются sell-stops LONG-позиций; за FL — buy-stops SHORT. Магнит-логика отличается от inefficiency: цена тянется не из-за "непроторговки", а из-за **collected stops** — институционалам нужна ликвидность.

## Свечи

`(2N+1)` последовательных свечей. Центр на индексе `N` (т.е. `N` слева + центр + `N` справа). Для canon `N=2` — 5 свечей.

## Направление и условия

### FH (Fractal High)

```
center.high > candles[i].high   ∀ i ≠ N    (strict)
```

Hi центральной свечи строго выше всех `2N` соседей. **Равенство недопустимо** (`>=` не считается).

### FL (Fractal Low)

```
center.low < candles[i].low   ∀ i ≠ N      (strict)
```

Зеркально.

### Невалидные случаи

- **Tied (==)**: хотя бы один сосед имеет равный `high` (для FH) или `low` (для FL) → `None`.
- **Both FH and FL одновременно**: редкий случай — центральная свеча имеет одновременно максимальный `high` и минимальный `low` среди всех (свеча "съедает" соседей). Geometrically такой случай ambiguous как swing point → возвращаем `None`.
- **Wrong length**: `len(candles) != 2N+1` → `None`.

## Подтверждение во времени

Фрактал **подтверждается только через `(N+1) * tf`** ms после `open_time` центральной свечи (для `N=2` это 3 бара). До этого момента — кандидат, который может быть переписан.

В smc-lib детектор не отслеживает время подтверждения сам — это ответственность вызывающего кода (как в существующем скрипте `~/smc-lib/scripts/plot_d_10_fractals_vwap.py`, где confirmed-фильтр применяется отдельно).

## Зона интереса = уровень

| Направление | **Зона интереса** | Геометрия |
|---|---|---|
| **FH** | `center.high` | **точка / уровень** (одно число) |
| **FL** | `center.low` | **точка / уровень** |

Это **отличается** от всех остальных primitive-элементов в smc-lib, которые имеют интервал. У fractal `zone` — это **price level**, не interval.

## Trading-модель (краткая)

🧲 За уровнем фрактала **скапливаются стопы** — это collected liquidity, магнит для цены.

| Сценарий взаимодействия | Что происходит | Интерпретация |
|---|---|---|
| **Sweep** | Цена пробивает уровень фитилём, закрывается обратно (1 свеча, не закрепляется) | **Главный SMC-сетап**: стопы собраны → ожидается разворот |
| **Закрепление пробоя** | 2+ closes за уровнем ([Правило 1](../../rules.md)) | Пробой удержан → продолжение пробойного движения |
| **Touch без пробоя** | Касание без захода за уровень | Слабая реакция; ликвидность не забрана |

**Сила фрактала** (см. [[feedback-fractal-liquidity-strength-and-sweep]]):
- `TF × возраст × cluster` — три фактора, формулы нет, контекстуально.
- Activity (touch count + volume) пока зона uncrossed — объективная метрика созревания (LuxAlgo Liquidity Swings).

**Sweep — TF-relative**: проверяется на том ТФ, где торгуешь сетап. HTF wick "проглатывает" LTF события — не агрегировать.

## Параметр N

| N | Окно | Где используется |
|---|---|---|
| **2** (default) | 5-bar Williams BW | canon smc-lib, `ob_liq.condition_3`, Pine WICK.ED 5-bar mode, `plot_d_10_fractals_vwap.py` |
| 1 | 3-bar | Pine WICK.ED 3-bar mode (менее строгий, чаще, шумнее) |
| ≥3 | (2N+1)-bar | Для HTF-only "глубоких" свингов |

## Эталонные примеры

### FH N=2 — synthetic (5 candles)

| Index | high | low |
|---|---|---|
| 0 (−2) | 102 | 95 |
| 1 (−1) | 104 | 97 |
| **2 (center)** | **110** | **99** |
| 3 (+1) | 108 | 100 |
| 4 (+2) | 105 | 98 |

- `center.high (110)` строго > всех соседей `{102, 104, 108, 105}` ✓ → **FH**
- `level = 110`

### FL N=2 — synthetic (5 candles)

| Index | high | low |
|---|---|---|
| 0 | 105 | 98 |
| 1 | 103 | 96 |
| **2 (center)** | 100 | **88** |
| 3 | 101 | 91 |
| 4 | 104 | 95 |

- `center.low (88)` строго < всех соседей `{98, 96, 91, 95}` ✓ → **FL**
- `level = 88`

### FH N=1 (3-bar) — synthetic

| Index | high | low |
|---|---|---|
| 0 | 100 | 95 |
| **1 (center)** | **105** | 98 |
| 2 | 102 | 96 |

- `center.high (105) > {100, 102}` ✓ → **FH**

## LuxAlgo Liquidity Swings — measurement layer (не primitive)

Индикатор LuxAlgo "Liquidity Swings" — это **measurement layer над FH/FL**, не определение FH/FL:
- Использует `N=14` для своих внутренних целей (29-bar pivots — более строгие swings).
- Измеряет **накопленный объём + touch count** пока зона uncrossed.
- Рисует ликвидность как **интервал фитиля** (`[body_top, high]` для FH zone).

Если когда-то понадобится strength-метрика — навешиваем сверху отдельным слоем, не вшиваем в primitive. См. [[feedback-fractal-liquidity-strength-and-sweep]].

## Связанные элементы

- `ob_liq` — condition №3 = Williams 5-bar LL/HH на свече `prev` (т.е. встроенный fractal-check c N=2).
- `rdrb` / `i_rdrb` — C2 displacement за `C1.low/high` эффективно = sweep mini-fractal.
- `snr` (не оформлено) — кластеризация близких FH/FL в горизонтальные зоны.
